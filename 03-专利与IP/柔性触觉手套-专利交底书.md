## 第二部分：技术交底书

### 一、本发明创造要解决的技术问题是什么？

针对现有技术中存在的以下缺陷，本发明提出如下技术问题：

1. **现有触觉传感器柔韧性差**：现有基于刚性 PCB 或柔性薄膜（如 PDMS/PI 基底）的压力传感器在反复弯折和贴合人体曲面时，存在耐疲劳性差、佩戴不适、透气性差等问题，难以满足长时间人手穿戴采集的需求。

2. **大面积阵列化与穿戴舒适性难以兼顾**：现有高密度触觉传感方案（如 32×32 以上阵列）多采用平面硬质或半硬质基底，无法在手套形态下实现大面积、高密度的传感点排布，且无法在手指关节等弯曲部位保持稳定的传感性能。

3. **缺乏面向人手工学的手套式触觉采集方案**：现有触觉采集设备多为单一传感点或小阵列（4×4/8×8），缺乏按照人手工学结构设计的手指关节分区 + 掌心大面积阵列的完整手套方案，无法精确捕捉人手操作时的全手指触觉分布数据。

4. **现有方案缺乏与虚拟空间可视化映射的对齐方法**：在将物理传感点的压力数据映射到虚拟三维手套模型时，现有方案未解决物理传感点与虚拟显示点之间的一一对应精准对齐问题，导致虚实位置偏差，影响数据可视化的准确性和后续数据分析的可靠性。

### 二、详细介绍背景技术

#### 2.1 触觉感知技术的发展趋势

随着人工智能、具身智能机器人和物理世界模型（World Model）技术的快速发展，触觉感知正在成为连接数字世界与物理世界的关键传感模态。传统的视觉（摄像头）和惯性（IMU）传感已无法满足高精度人机交互、机器人灵巧操作和物理仿真建模的需求，触觉数据的重要性日益凸显。

在机器人领域，灵巧手的精细操作（如抓取易碎物品、进行精密装配）需要实时的触觉反馈；在人机协同场景中，设备需要感知人的操作力度和接触位置以做出恰当的响应。同时，基于物理引擎（如 MuJoCo）的仿真训练正在大量依赖真实触觉数据进行 Sim-to-Real 迁移。触觉采集将成为未来机器人训练和物理 AI 的基础数据基础设施。

#### 2.2 现有触觉传感技术方案

**（1）基于刚性基底的压力传感器**

以 MEMS 工艺制造的硅基压阻/电容式传感器为代表。优点：精度高、响应快、标定稳定。缺点：刚性不可弯折、不适合穿戴、单点成本高、大面积阵列化困难。典型产品包括帕西尼（Pasin）的触觉传感器等。

**（2）基于柔性薄膜基底的传感器**

以 PDMS（聚二甲基硅氧烷）、PI（聚酰亚胺）等柔性高分子材料为基底，通过印刷或蒸镀工艺制备电极和传感层。代表研究机构包括华中科技大学（38 件相关专利）、华南理工大学（32 件相关专利）等。优点：可弯折、厚度薄。缺点：透气性差、长时间佩戴不适、大面积制备时均匀性难以保证。

**（3）基于织物基底的柔性传感阵列**

以织物为基底，通过镀银纱线/导电纱线编织形成传感阵列。代表企业包括钜侨工业、上海尧乐科技等。目前市面上已有的织物传感方案多面向床垫压力监测、坐垫压力分布等相对静态的大面积应用场景，缺乏针对人手操作动态场景的、按人手工学设计的分区阵列手套方案。

#### 2.3 触觉手套类产品现状

目前市场上的触觉手套产品主要包括：
- **数据手套（如 CyberGlove）**：基于光纤弯曲传感器或惯性传感器，检测手指弯曲角度和姿态，**缺乏压力/力触觉检测能力**。
- **Haptic 反馈手套（如 HaptX）**：侧重于力反馈输出，而非触觉数据采集。
- **单点式触觉手套**：在指尖等关键位置安置少量压力传感点，无法实现全手掌尺度的压力分布图。

现有技术中，将全柔性织物压力传感阵列以手套形态呈现，并按手指关节生物学结构（近端指节、中间指节、远端指节）和掌心区域分别设计传感点阵列的完整方案，尚未在已公开的专利或产品中被披露。

### 三、现有技术的缺点是什么？针对这些缺点，说明本发明的目的

| 现有技术缺点 | 本发明的目的 |
|:------------|:-----------|
| 触觉传感器刚性强、不可穿戴 | 提供一种全柔性织物基底的、可穿戴的触觉手套，实现长时间舒适佩戴 |
| 薄膜基底不透气、佩戴不适 | 采用全织物结构，天然透气、柔软亲肤 |
| 现有手套方案缺乏压力检测 | 实现高密度压力分布传感，覆盖手指全部关节面和掌心 |
| 缺乏大的传感点布局 | 手指采用 2×2 子阵列按关节划分，掌心采用 8×8 网格阵列，总计 124 个传感点 |
| 现有方案虚实映射存在偏差 | 提供一种物理传感点与虚拟显示点之间的精准对齐映射方法 |
| 缺乏传感器到机器人的适配 | 采用通用数据接口，适配不同厂商机器人平台的软硬件接口 |

#### 4.2 柔性织物传感手套结构

**4.2.1 传感单元材料结构**

传感单元采用三层织物结构：

- **上层电极层**：镀银纱线，通过刺绣工艺形成锯齿形/平线走线（横向排列），每根纱线对应一行扫描行
- **中间传感层**：压敏纱线，以尼龙为基底涂覆碳黑导电涂层，受压时接触电阻减小
- **下层电极层**：镀银纱线，通过刺绣工艺形成锯齿形/平线走线（纵向排列），每根纱线对应一列扫描列

上下层电极正交排列，交叉点即为传感单元节点。当外部压力作用于传感点时，中间压敏纱线层形变，上下层电极之间的接触电阻降低，形成可测量的电阻变化。

**4.2.2 手套传感点排布设计**

本发明采用按人手生物学结构优化的分区排布方案：

**手指区域**（5 指 × 3 关节 = 60 个传感点）：
- 每根手指按解剖学划分为：近端指节（PIP）、中间指节（MIP）、远端指节（DIP）
- 每个指节区域布置一组 **2 × 2 传感点阵列**（4 个传感点）
- 传感点沿手指纵向均匀排布，覆盖手指操作时的主要施力面和受力区域

**掌心区域**（1 个区域 = 64 个传感点）：
- 掌心布置 **8 × 8 网格阵列**
- 阵列覆盖大鱼际区、小鱼际区和掌心中央区
- 传感点间距根据掌心曲面曲率优化，确保不同手型和不同手持姿势时的接触覆盖

**全手套总计**：5 指 × 3 关节 × 4 点/关节 + 64 点/掌心 = **60 + 64 = 124 个传感点**

（可选实施方式中，可扩展为 6指方案，此时总计 6×3×4+64 = 136 个传感点。）

**4.2.3 布线方案**

所有行线和列线通过柔性布线汇聚至手套手腕部，通过 **FPC（柔性印刷电路）排线**引出至外部信号采集电路板。这种汇聚走线方式确保：
- 手指弯曲时行/列线随织物自然弯折，不影响信号传输
- 手腕部单点引出，佩戴时不妨碍手指自由活动
- FPC 连接器插入采集板，便于手套更换和清洗

#### 4.3 信号采集电路

**4.3.1 电路架构**

信号采集电路板置于手腕处（可设计为腕表形态），包含：

1. **多路复用选通模块**：对 124 个传感点进行行列扫描选通，逐点测量
2. **模拟前端调理模块**：包含运算放大器和低通滤波器，对电阻信号进行调理
3. **模数转换模块**：12-bit ADC，将模拟电压信号转换为数字量（分辨率 4096 级）
4. **微控制器模块**：控制扫描时序、数据打包和通用数字通信接口发送
5. **电源管理模块**：通过 USB 供电（5V），为各模块提供稳定电压

**4.3.2 预留扩展接口**

采集电路板采用模块化设计，预留了多处扩展接口：

- **预留 IMU 芯片焊接位**（I²C/SPI 接口，含中断引脚），未来可集成 9 轴惯性测量单元实现手部姿态同步采集
- **预留摄像头触发同步接口**（GPIO 带计时器输入捕获），未来可配合摄像头实现视觉-触觉时序对齐
- **预留无线通信模块接口**（支持蓝牙/BLE/Wi-Fi 模块），未来可实现无线数据传输
- **预留 AI 协处理器接口**（SPI+中断线），未来可部署轻量级神经网络进行实时触觉特征提取
- 所有预留接口均已预先布线、预留供电和去耦电容位置，后期焊接即用

> 上述预留接口设计确保本发明的采集板具有极强的可扩展性，后续可根据应用场景灵活增加多模态传感功能，而无需重新设计电路板。

#### 4.4 数据处理与虚实映射方法

**4.4.1 通用数字通信接口**

上位机与采集板之间采用通用数字通信接口连接，支持但不限于以下通信模式：
- 有线模式：USB 虚拟串口（CDC）、USB HID、以太网等
- 无线模式：蓝牙 BLE、Wi-Fi（通过预留接口扩展模块实现）

数据协议采用 JSON 格式，包含：时间戳、各传感点行列索引和对应 ADC 原始值/压力映射值。

**接口设计采用驱动层抽象架构**，通过标准化的数据格式和指令集，可适配不同厂商机器人平台的软硬件接口（如 ROS/ROS2 消息格式、CAN 总线数据帧、Modbus 协议等），仅需在上位机驱动层做协议转换，无需改动硬件和固件。

**4.4.2 物理-虚拟空间坐标映射方法**

本发明提出的虚实映射方法包含以下步骤：

1. **物理传感点标定**：对每个传感点进行物理位置编号，形成 (行索引, 列索引, 手指编号, 关节编号) 的四维坐标矩阵

2. **虚拟手套模型建立**：建立三维虚拟手模型，模型表面按相同拓扑结构划分为对应的网格节点（5指×3关节×4点 + 掌心64点）

3. **索引对齐映射表**：建立物理传感点索引与虚拟模型网格节点索引之间的一一对应关系表。例如：
   - 物理点 `(手指1, 关节1, 行0, 列0)` ↔ 虚拟节点 `(拇指, 近端指节, 左上角)`
   - 物理点 `(掌心, 行3, 列5)` ↔ 虚拟节点 `(掌心, 第三行第五列)`

4. **实时数据渲染**：微控制器以 15Hz（可调）帧率采集全阵列数据，通过通用数字通信接口传输至上位机。上位机按照映射表将压力值逐点对应到虚拟手套模型的对应网格节点，实现物理压力分布到虚拟显示的**逐点一对一精准映射**

5. **可视化颜色编码**：将 ADC 值（0-4095）映射到预设的颜色渐变方案（如蓝色→绿色→黄色→红色），在虚拟手套模型上实时渲染压力热力图

该映射方法的关键创新在于：**物理传感点与虚拟显示节点之间通过显式的索引对齐映射表保持严格的一一对应关系**，而非通过空间位置插值或区域近似，从而确保每一个物理传感点采集的压力数据都能精确对应到虚拟模型的指定位置，不产生位置偏移或数据窜位。

#### 4.5 工作流程

1. 操作者佩戴触觉手套，进行各种手部操作（如抓取物体、操作工具、进行化学实验等）
2. 采集板以 15Hz 频率逐行扫描 124 个传感点，获取各点压力数据
3. ADC 将模拟信号转换为数字量，MCU 打包为 JSON 数据帧
4. 数据通过通用数字通信接口发送至上位机
5. 上位机接收数据后，按照虚实映射表将各传感点压力值精确映射到虚拟手套模型对应位置
6. 实时渲染压力热力图，同时将数据存储为标准化格式供后续分析
7. （可选）通过驱动层抽象适配接口，将数据转发至机器人控制系统或仿真引擎

### 五、本发明的关键点和保护点是什么？

按照重要次序逐条列出：

**保护点 1：全柔性织物触觉手套的手工学传感点布局方案**
采用织物基底传感阵列，按人手工学结构设计传感点布局：手指分 5（或 6）指 × 3 关节，每个关节区域布置 2×2 传感点子阵列，掌心布置 8×8 传感阵列，构成覆盖全手主要施力面的高密度触觉采集系统。该布局方案在保证高分辨率触觉采集的同时，实现了织物基底的柔性可穿戴和透气舒适性。

**保护点 2：物理传感点与虚拟显示点之间的一一对应精准映射方法**
建立物理传感点索引矩阵与虚拟手模型节点索引矩阵之间的显式索引对齐映射表，实现每个物理传感点与虚拟模型对应节点之间的逐点一对一定位映射，而非通过空间插值或区域近似。确保触觉数据在虚拟空间中精确还原物理位置，消除虚实位置偏差。

**保护点 3：预留多模态扩展接口的模块化采集电路板**
采集电路板除包含基本的 MUX-ADC-MCU 信号采集链路外，还预留了 IMU 芯片焊接位、摄像头同步触发接口、无线通信模块接口和 AI 协处理器接口。所有预留接口均已完成布线、供电和信号引脚预留，后期可直接焊接扩展模块实现多模态传感功能，无需重新设计电路板。

**保护点 4：通用数字通信接口与多平台适配架构**
采集电路与上位机之间采用通用数字通信接口（不限定具体物理层和协议层），通过上位机驱动层抽象设计，将标准化数据格式转换为不同目标平台所需的协议格式（包括但不限于 ROS/ROS2 消息、CAN 总线帧、Modbus、自定义协议等），实现与不同厂商机器人平台和控制系统的即插即用式数据对接。

**保护点 5：三层织物传感单元结构 + 手套形态集成方案**
以镀银纱线为上下电极层、压敏纱线为中间传感层的三层织物传感单元，通过 FPC 排线将全部行/列线汇聚至手腕部引出，与信号采集板通过可插拔连接器连接。这种结构在保证传感性能的同时，实现了传感单元与手套本体的柔性集成，便于手套的更换和清洗。

### 六、与第二部分所属的最好的现有技术相比，本发明有何优点（有益效果）？

| 对比维度 | 现有技术 | 本发明 | 有益效果 |
|:--------:|:---------:|:-------:|:---------|
| **可穿戴性** | 刚性/薄膜基底，不透气 | 全织物结构，天然透气 | 可长时间佩戴不闷汗，适合实际操作场景数据采集 |
| **传感点密度** | 少量离散点布局 | 124 点（5指 × 3关节 × 4点/关节 + 掌心 8×8） | 首次实现对人手操作的精细压力分布全貌采集 |
| **灵活性** | 手指弯曲时薄膜基底受限 | 织物随手指自然弯折 | 不妨碍人手正常操作，采集数据更接近真实操作 |
| **虚实映射精度** | 无标准映射方法或采用近似 | 一一对应的索引对齐映射 | 数据准确还原物理位置，无偏移 |

### 七、本发明创造的替代方案

**7.1 传感走线工艺替代**
- **方案 A（优选）**：锯齿形刺绣走线（母技术基线方案），增加电极-传感层接触面积，提高信号稳定性
- **方案 B（替代）**：平线走线（本手套验证方案），工艺更简洁，适合大面积快速制备
- **方案 C**：导电织物裁剪为电极条带后缝制贴合，无需刺绣设备，手工可制作

**7.2 手指数量变体**
- **方案 A（5指方案）**：标准人手工学设计，60 指节点 + 64 掌心点
- **方案 B（6指方案）**：增加小指辅助关节/额外传感覆盖，72 指节点 + 64 掌心点

**7.3 阵列密度变体**
- **标准密度**：2×2 每关节 + 8×8 掌心（共 124 点）
- **高密度版**：3×3 或 4×4 每关节，12×12 掌心（阵列数可扩展至 200+ 点）
- **低密度版**：1×1 每关节，4×4 掌心（简化方案，降低成本）

**7.4 通信方案替代**
- **有线通信**：USB 虚拟串口、USB HID、以太网
- **无线通信**：蓝牙 BLE、Wi-Fi（通过预留接口扩展）
- **工业总线**：CAN/CAN-FD、RS-485（通过驱动层适配）

**7.5 传感原理替代**
- **本方案（压阻式）**：三层织物电阻变化检测
- **替代方案 A**：电容式织物传感（上下电极+介电层）
- **替代方案 B**：压电式织物传感（PVDF 等压电材料嵌入）
- **替代方案 C**：摩擦电式传感（TENG 原理，自供能）

**7.6 传感器数量变体**
- **全指方案**：5 指全覆盖（完整方案）
- **精简方案**：仅拇指+食指+中指（3 指方案，适合简单抓取操作）
- **仅掌心方案**：仅保留掌心 8×8 阵列（适合坐姿压力监测）
