## 二、三阶段落地计划（融资版 vs 真实版对照）

### 阶段一：MVP 数字孪生 + 示教复现（0-3个月）
**融资话说：** "构建实验室数字孪生底座，实现远程遥操作与数据资产沉淀"

**实际做：**

| 模块 | 做啥 | 难度 |
|------|------|------|
| ① **数字孪生** | 用 Unity/Unreal 建 1:1 浙大实验室 3D 场景，摄像头画面+传感器数据实时映射到虚拟空间 | ⭐⭐ 有3D建模师就能做 |
| ② **遥操作手套** | 触觉手套缝制成型 → 124点信号稳定 → 映射到虚拟手/机器人手同步运动 | ⭐⭐ 已有基础 |
| ③ **单操作演示** | 研究员戴手套做一次"倒液" → 记录轨迹 → 虚拟空间回放 → 机械臂复现 | ⭐⭐⭐ |
| ④ **大屏可视化** | 数字孪生大屏：左边真实验室摄像头，右边虚拟实验室同步动，中间数据面板飞 | ⭐ 前端活 |

**交付物（融资 Demo）：**
> 研究员在浙大实验室戴上手套操作 → 屏幕上数字孪生实时同步 → 记录操作数据 → 机械臂复现同一动作
> → **"从人到机器，从物理到数字的完整闭环"**

**融资包装关键词：**
- 数字孪生实时映射
- 遥操作与示教学习
- 实验操作数据资产化

**真实投入：** 1个Unity开发 + 你调试手套 + 浙大专一个实验台位
**预算估算：** ~10-15万（外包3D建模+开发）

## 三、数字孪生的真实定位（融资话术 vs 技术事实）

既然你要用数字孪生做融资亮点，这里告诉你**数字孪生在这个项目里到底能做什么、不能做什么：**

### ✅ 真的能做（放在路演里讲）

```
1. 实时可视化管理
   管理者打开大屏就能看到浙大实验室当前状态
   "出差也能看实验室在做什么实验"

2. 操作数据回放
   记录研究员每次操作的完整轨迹
   "每个实验动作都可追溯、可复现"

3. 仿真推演
   在虚拟环境里测试新实验方案
   "先在数字孪生里跑一遍，再在真实实验室执行"
   (这个其实需要通过MuJoCo才能做到，数字孪生只是入口)
```

### ⚠️ 听起来高级但技术上是真的

```
"数字孪生驱动的仿真训练" 
→ 真相：数字孪生负责展示和映射，MuJoCo负责物理仿真
→ 但在融资时讲成一句话："数字孪生环境里训练AI模型" 
   懂行的知道这是两个系统，但不妨碍你这么说

"实时数字孪生镜像"
→ 真相：摄像头画面+传感器数据刷新，延迟<100ms
→ 技术上不复杂，但视觉效果震撼
```

### ❌ 不能吹的（吹了会被技术尽调打脸）

```
- "数字孪生自动生成实验方案" → 这是LLM的事
- "数字孪生实时优化实验参数" → 表征数据反馈才能做，需要完整的闭环

这两个不要混为一谈，资方请的技术顾问一眼就能识破
```

## 四、关键风险与实话实说

| 风险 | 真实难度 | 应对 |
|------|---------|------|
| 触觉手套稳定性 | ⭐⭐ | 缝制后多测试，不行就缩小传感点数 |
| 机械臂精度不够 | ⭐⭐⭐ | 化学实验需要亚毫米级，选FRANKA或UR |
| LabVLA接入触觉 | ⭐⭐⭐⭐ | 这部分最不确定，T-Rex论文可参考但实现难度大 |
| 催化剂特殊性 | ⭐⭐⭐⭐ | 涉及液体/粉末/高温，不同于通用操作 |
| 浙大配合节奏 | ⭐⭐ | 明确分工边界，避免互相等 |
