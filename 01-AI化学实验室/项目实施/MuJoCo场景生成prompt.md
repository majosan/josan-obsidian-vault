# Prompt 3 — MuJoCo 虚拟人手场景搭建

## 状态
- 时间：2026-05-25
- 交付物：`scene.xml` + `load_scene.py`
- 前置条件：Phase 1 完成（IMU + 摄像头数据能独立输出）

## 提示词

请帮我创建一个 MuJoCo 仿真场景，包含以下内容：

一、场景元素
1. 一张桌面（1m × 0.6m × 0.02m 的平板，离地 0.7m）
2. 一个烧杯（圆柱体，直径 6cm，高 8cm，放在桌面左前方）
3. 一个量筒（圆柱体，直径 3cm，高 12cm，放在桌面右前方）
4. 一个绿色圆球（直径 4cm，放在桌面中间）
5. 一个人手模型（见下文骨骼结构）

二、人手模型骨骼树（请用 MJCF XML 文件描述）
- 手腕：球关节（3 自由度旋转），固定在桌面上方约 30cm 处
- 手掌：box geom，尺寸 0.04 × 0.04 × 0.02
- 5 根手指，每根 3 个指节，使用 capsule geom
- 每个指节一个 hinge 关节（绕 Z 轴），控制弯曲
- 指尖加 touch sensor site（球体，半径 5mm）
- 具体尺寸比例参考真实成人手掌

三、摄像头视角
- 默认第三人称俯视视角（桌面正上方 45° 斜角）
- 同时开一个子窗口显示"人手第一人称"视角

四、控制方式
- 通过 Python API 设置：
  - qpos[0:4] = 手腕四元数（球关节）
  - qpos[4:19] = 15 个指关节角度
  - 共 4 + 15 = 19 个自由度
- 使用 mj_forward() 推进，不跑物理动力学（仅用于可视化同步）

请生成一个 complete_scene.xml 文件和配套的 load_scene.py，
load_scene.py 要求能接收 JSON 格式的传感器数据帧并刷新渲染画面。

## 验证标准
- [ ] `python load_scene.py` 打开 MuJoCo 窗口，显示桌面+道具+虚拟手
- [ ] 手动修改代码中的 qpos 参数，虚拟手做出对应动作
- [ ] 烧杯/量筒/球在正确位置
