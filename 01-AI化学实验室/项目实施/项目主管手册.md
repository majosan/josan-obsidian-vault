## 三、三阶段训练详解

### 总体流程

```
RoboGenesis → 海量仿真数据 → ① FAST预训练
                                   │
                                   ▼
                       ② Flow Matching后训练
                                   │
                                   ▼  + 真实数据（手套+IMU+视觉）
                       ③ 任务微调 → 部署到机器人
```

### 阶段①：FAST预训练

| 项目 | 内容 |
|------|------|
| **用啥数据** | RoboGenesis 生成的仿真数据（10万+条轨迹） |
| **在哪跑** | PyTorch，MuJoCo只是环境反馈接口 |
| **学什么** | 动作的「大致分布」——比如倒水大概是什么弧线 |
| **输出** | 一个有基础能力的模型（动作粗糙但方向对） |
| **类比** | 学画画先画1000个圆的轮廓，不要求圆 |

### 阶段②：Flow Matching后训练

| 项目 | 内容 |
|------|------|
| **用啥数据** | 和预训练**同一批**仿真数据 |
| **换啥方法** | 从"学分布"变成"精炼质量"——Flow Matching技术 |
| **还干啥** | 知识隔离——记住预训练学的东西，不被冲掉 |
| **输出** | 动作更流畅、更精准的模型 |
| **类比** | 拿着范本练习，把圆画得又圆又顺 |

### 阶段③：任务微调

| 项目 | 内容 |
|------|------|
| **用啥数据** | 真实采集数据（触觉手套+摄像头+IMU） |
| **量级** | 几百到几千条，远比仿真少 |
| **干啥** | 弥合仿真和现实之间的差距（Sim-to-Real Gap） |
| **输出** | 能部署到真实机器人的最终模型 |
| **类比** | 在驾校练熟了（仿真），上路练几天（微调），就能开了 |

### 核心理解

```
预训练（数据量大、便宜）：仿真数据  ← RoboGenesis
后训练（数据量相同、方法不同）：仿真数据精炼
微调（数据量小、贵）：真实数据     ← 手套+摄像头
```

## 四、所有名词一句话解释

### 仿真与工具

**MuJoCo** — 免费开源的物理引擎，计算重力、碰撞、摩擦、关节运动。不提供华丽界面，只算物理。比你想象得轻量，能跑在 MacBook 上。

**MJX** — MuJoCo 的 GPU 加速版。能在 TPU/GPU 上同时跑上百万帧，大规模并行训练用的。

**RoboGenesis** — 跑在 MuJoCo 之上的数据工厂。封装了 pick（抓）、pour（倒）、stir（搅拌）等 11 种原子操作。你只需要说"生成 1000 条倒水数据"，它就自动在虚拟实验室里倒 1000 次，记录每一步动作。

**Genesis（另一个项目）** — 关注中的备选仿真引擎，原生支持流体和变形体模拟。v0.3.0 还比较新，2027 年再评估是否迁移。

**NVIDIA Isaac Sim** — 完整的一站式仿真平台（不是单纯引擎），有顶级画质、传感器模拟、域随机化。但太重，PC 跑不动，目前不适合你。

### 模型与框架

**LabVLA** — 浙大+上海AI Lab开源的实验室 VLA 模型。核心是 Qwen3-VL-4B 视觉语言模型 + DiT 动作专家。你的主参考框架，代码全开源。

**VLA** — Vision-Language-Action 的缩写。输入图+话，输出动作。VLA 是**近期可行路线**。

**世界模型** — 能预测物理世界变化的模型。你推烧杯，它知道烧杯会往哪倒。比 VLA 难得多，是**远期目标**。

**DiT** — Diffusion Transformer。LabVLA 里用来生成连续动作的模块，把动作当做"图像"来生成。

**Qwen3-VL-4B** — 阿里通义千问的视觉语言模型，LabVLA 用它来理解画面和指令。4B=40亿参数。

**LLM** — 大语言模型。你的项目里浙大团队用它生成实验方案/配方。

**Foundation Model** — 基础模型，在大量数据上预训练后可以适配各种下游任务。LabVLA 理论上就是一个实验室领域的 Foundation Model。

### 编码与数据

**编码通道** — 把原始传感器数据转成模型能理解的格式。你的手套数据必须经过这一步才能喂给模型。

**CNN（卷积神经网络）** — 空间编码器。把 124 个压力点排成 8×16 的矩阵，CNN 看"压力集中在哪个区域"。

**VQ-VAE（向量量化变分自编码器）** — 时间压缩器。把连续 16 帧的趋势（1984 个数字）压缩成 11 个离散 token。压缩比约 169:1。

**压缩比 169:1** — 原始 1984 个数字 → 11 个 token。不是丢掉信息，而是提取了关键模式。

**域随机化** — 训练时随机改虚拟场景的颜色、光照、纹理、物体位置。让模型学到"物体的共性"而不是"红色的桌子"，提升仿真到现实的迁移率。

### 概念与方法论

**Sim-to-Real** — 仿真训练→现实部署的整套方法论。包含域随机化、系统辨识、噪声注入等技术。

**数字孪生 (Digital Twin)** — 物理实体的实时虚拟镜像。摄像头+传感器数据实时同步到 3D 虚拟场景。本质是**展示层**，不是训练工具。

**示教学习 (Imitation Learning / LfD)** — 人做一遍操作，记录轨迹，模型直接模仿。你的触觉手套就是这个用途——示教器/数据捕捉器。

**知识隔离** — 训练时保护已学知识不被新数据冲掉。后训练和微调阶段必须用的技术。

**Flow Matching** — 一种生成式训练方法，把粗糙的动作分布精炼成流畅精准的动作。后训练阶段使用。

**Reality Gap（现实差距）** — 仿真和真实世界之间的差异。摩擦力算不准、光照不一样、传感器有噪声。微调和域随机化都是为了弥合这个 gap。

**原子技能 (Atomic Skill)** — RoboGenesis 里的最小操作单元。pick（抓）、pour（倒）、press（按）等每个都是带多阶段状态机的原子技能。

### 硬件

**触觉手套** — 你的核心数据采集设备。124 个传感点，缝在手套内侧。角色是**示教器**，不是机器人皮肤。采集研究员的操作数据给模型学。

**IMU** — 惯性测量单元（加速度+陀螺仪+磁力计）。贴在手背测手部姿态，9-DOF 出四元数。

**RealSense D435** — Intel 深度摄像头。提供毫米级实测深度，比单目 RGB 估算的 Z 轴更准确。

**Franka Emika** — 高精度科研级机械臂，7 自由度，力矩传感器每个关节都带。适合实验室精细操作，但贵。

**UR5e** — 优傲机器人出品，工业级协作臂，6 自由度。更稳定，性价比更高，精度略低于 Franka。

## 五、各环节难度评估

### 🥇 最难：触觉编码通道接入 VLA（风险等级：🔴🔴🔴🔴🔴）

| 维度 | 说明 |
|------|------|
| **为什么难** | 全世界没人做过织物124点手套 + VLA 的组合 |
| **具体卡点** | ① 频率不匹配（视觉5fps vs 触觉15Hz）|
| | ② VQ-VAE 编码器训练本身需要大量数据 |
| | ③ 融合时机选错了反而降性能（T-Rex 的教训：17%→6%）|
| **你的对策** | 这个环节投入最多精力盯着，判断进展 |

### 🥈 第二难：Sim-to-Real 迁移（风险等级：🔴🔴🔴🔴）

| 维度 | 说明 |
|------|------|
| **为什么难** | 催化剂实验涉及液体/粉末/高温，不只是抓杯子 |
| **具体卡点** | ① RoboGenesis 流体模拟很初级 |
| | ② 粉末称量在仿真里几乎没有成熟方案 |
| | ③ 一毫米误差在化学实验里就是失败 |
| **你的对策** | 前期演示只做刚体操作，液体阶段再攻坚 |

### 🥉 第三难：全流程可靠性（风险等级：🔴🔴🔴🔴）

| 维度 | 说明 |
|------|------|
| **为什么难** | Demo 翻车的后果和论文翻车不一样 |
| **具体卡点** | ① 80% 准确率在论文叫 SOTA，在 Demo 叫翻车 |
| | ② 异常处理（抓空、倒洒）比正常流程还难做 |
| | ③ 机械臂长期运行精度会漂移 |
| **你的对策** | 融资演示前反复排练，准备"安全版"脚本 |

### 其他环节难度

| 环节 | 难度 | 说明 |
|------|------|------|
| 数字孪生大屏 | ⭐ | 3D建模+前端，外包搞定 |
| 手套硬件稳定 | ⭐⭐ | 缝制+信号调试，纯工程活 |
| MuJoCo 场景搭建 | ⭐⭐ | 有操作文档，花时间就能搭 |
| RoboGenesis 跑通 | ⭐⭐⭐ | 有文档有代码，需要配置环境 |
| LabVLA 部署 | ⭐⭐⭐ | 有开源代码，需要 GPU 服务器 |
| 机械臂选型集成 | ⭐⭐⭐ | 选对型号，基本配置就好 |
| 自定义原子技能 | ⭐⭐⭐⭐ | 旋转瓶盖和擦拭需要自己设计状态机 |
| 全流程自动化 | ⭐⭐⭐⭐⭐ | 多臂协同+异常处理+表征反馈 |

```

   → 能拿白板画清楚区别：VLA=直接输出动作，世界模型=先推演再行动
   → 为什么你选VLA：成熟、有开源、能快速出Demo

   → 预训练用啥数据、后训练干嘛、微调干嘛

3. 数字孪生 vs 仿真 vs 数字镜像
   → 数字孪生=展示层（融资用）
   → MuJoCo=训练层（技术用）
   → 两个可以一起讲但不能混为一谈

4. 触觉编码基本原理
   → CNN 处理空间压力分布
   → VQ-VAE 压缩时间序列
   → 为什么压缩比 169:1 还不丢信息

5. 硬件选型核心参数
   → 机械臂：精度/自由度/负载/接口
   → 摄像头：深度/帧率/视场角
   → GPU：显存/算力/功耗
```

## 八、常见误区与红线

### 常见认知误区

| 误区 | 真相 |
|------|------|
| "模型是在 MuJoCo 里跑出来的" | ❌ 模型在 PyTorch 里训练，MuJoCo 只提供环境反馈 |
| "RoboGenesis 是个模型" | ❌ 它是数据生成器，不是模型 |
| "数字孪生就是仿真" | ❌ 数字孪生=实时镜像，仿真=物理计算 |
| "预训练用真实数据" | ❌ 预训练用仿真数据（量大便宜），微调用真实数据（少而贵） |
| "VLA 能做世界模型的事" | ❌ VLA 直接出动作，世界模型要先推演再决策 |
| "模型训练完还在 MuJoCo 里" | ❌ 训练完导出权重文件，部署到真机，MuJoCo 退役 |
| "触觉数据直接喂给模型" | ❌ 必须经过编码通道转化，否则模型看不懂 |
| "Sim-to-Real 是一个工具" | ❌ 是一套方法论，不是某个软件 |

### 🔴 融资红线（绝对不能吹的）

```
❌ "数字孪生自动生成实验方案"
   → 这是 LLM 的事，数字孪生只是可视化

❌ "数字孪生实时优化实验参数"
   → 需要完整闭环（表征数据反馈），短期内做不到

❌ "我们自研了世界模型"
   → 你走的是 VLA 路线，世界模型是远期目标

❌ "完全无人化"
   → 目前的人机协同是 1人+多机器人，不是全无人

❌ "任何化学实验都能做"
   → 聚焦催化剂合成，限定场景
```
