# 双臂递物训练方案（RL 强化学习）

> 创建日期：2026-06-08
> 目标：让机器人通过强化学习自主学会"左臂抓起 → 悬空 → 右臂接过 → 放到新位置 → 回原点"

---

## 一、任务定义

### 场景设定

```
                ┌──────── 实验台 ────────┐
                │                          │
   左臂初始位     │     [烧杯]              │  右臂初始位
   (home_left)   │     初始位置              │  (home_right)
        🦾      │                          │      🦾
                │                          │
                │          [目标位置]       │
                │                            │
                └──────────────────────────┘
```

### 动作流程（一个完整 episode）

```
Step 0: 双臂在 home 位，烧杯在桌面上初始位置
    │
Step 1: 左臂伸向烧杯 → 夹爪闭合 → 抓起 → 提升悬空
    │
Step 2: 右臂伸向悬空的烧杯 → 夹爪闭合 → 接过
    │
Step 3: 左臂松开 → 缩回 home_left
    │
Step 4: 右臂带着烧杯移动到目标位置 → 放到桌面 → 松开
    │
Step 5: 右臂缩回 home_right → episode 结束
```

### 成功标准
- ✅ 烧杯不掉落
- ✅ 左臂抓到后稳定提起（距离桌面 ≥ 10cm）
- ✅ 右臂成功接过（不碰掉）
- ✅ 放到目标位置 ±2cm 内
- ✅ 双臂回到 home 位

## 二、推荐训练策略：课程学习（Curriculum Learning）

这个任务对单策略来说太复杂（14 个关节 + 2 个夹爪同时协调），**不能一步到位**。需要用**课程式学习**——从简单子任务开始，逐步叠加难度。

### 训练阶梯

```
Level 1 ─── 左臂独立 Pick-and-Place（已经跑通）
   │         抓烧杯 → 放到目标位 → 回 home
   │         目标：掌握基础的抓取-放置能力
   │
Level 2 ─── 右臂独立 Pick-and-Place
   │         对称训练，让右臂也学会
   │
   ▼
Level 3 ─── 左臂抓取 + 提升悬空（hold）
   │         不要求放回，只要求抓住提起并保持
   │         目标：掌握"稳定悬停"技能
   │
   ▼
Level 4 ─── 右臂从静止目标物接取
   │         左臂被固定住（torque off），只当架子
   │         右臂学会从固定位置接过物体
   │         目标：拆解"接物"这个子技能
   │
   ▼
Level 5 ─── 双臂联合：左臂抓+悬空，右臂接过
   │         左臂主动悬停，右臂伸过来接
   │         关键难点：两臂不能互撞
   │
   ▼
Level 6 ─── 全流程：抓→悬空→接过→放→回原点
   │         完整 5 步动作
   │         目标：成功率 ≥ 80%
   │
   ▼
Level 7 ─── 泛化（可选）
            随机化烧杯初始位置
            随机化目标位置
            不同形状的器皿
```

### 为什么不能直接跳到 Level 6

```
直接端到端训练的问题：
  ┌──────────────────────────────┐
  │ 每次 episode 要完成 5 步      │
  │ 长达 ~500 个仿真步           │
  │ 一步失败（烧杯掉）= 整局失败   │
  │ reward 信号极其稀疏           │
  │ 几乎每次都是 0 reward         │
  │ → 策略根本学不到东西          │
  └──────────────────────────────┘

课程学习的优势：
  ┌──────────────────────────────┐
  │ Level 1 的 reward 密集        │
  │ 每接近烧杯 1cm → 有奖励      │
  │ 学得快，验证快                │
  │ 每个子技能作为下一个 level    │
  │ 的初始化权重（参数迁移）      │
  │ → 最终策略质量高              │
  └──────────────────────────────┘
```

## 三、动作空间（Action Space）

```
总维度：7（左臂关节）+ 7（右臂关节）+ 1（左夹爪）+ 1（右夹爪）= 16 维

每臂：6 或 7 个关节位置（POSITION MODE） + 力矩控制（TORQUE MODE）

注意：
- OpenArm 的 MJCF 使用 torque control（力矩控制）
  这意味着 action 输出的是力矩，而不是位置
  对于 RL 来说稍微难一点（力矩→位置需要经过物理引擎）
  如果做 position control 更稳定，可以先改 actuator 为 position mode

- ALOHA（你现在跑的）使用 position control
  所以如果先在 ALOHA 上跑通 Level 1-3，
  迁移到 OpenArm 时需要改控制模式
```

### 观测空间（Observation Space）

```
双臂关节角度         × 14（6+6 或 7+7）
双臂关节角速度       × 14
夹爪开合状态          × 2
烧杯位置（xyz）       × 3
烧杯速度（xyz）       × 3
目标位置（xyz）       × 3
左夹爪到烧杯距离      × 1
右夹爪到烧杯距离      × 1
→ 约 41 维观测
```

## 四、奖励函数设计（Reward Shaping）

**这是整个训练成败的关键。** 奖励设计决定了策略往哪个方向学。

### Level 1-2（单臂 Pick-and-Place）

```
每步奖励 = 
  - 接近物体奖励：-||gripper - object|| × w₁
  + 抓取成功奖励：+10（物体随夹爪移动时触发）
  - 运输奖励：-||object - target|| × w₂
  + 放置成功奖励：+20（物体在目标位 ±2cm）
  - 掉落惩罚：-5
  - 时间惩罚：-0.1/步
```

### Level 3（悬空 Hold）

```
在前面的基础上改：
  + 提升奖励：(object_z - table_z) × w₃（越高越好）
  + 稳定奖励：-||object_velocity|| × w₄（越稳越好）
  - 去掉放置成功奖励（不要求放）
```

### Level 4（从静止接物）

```
  - 右夹爪接近物体奖励 × w₁
  + 右爪成功闭合触发奖励 +10
  + 物体随右爪移动 +5
  - 掉落惩罚 -10（此时左臂不参与）
  
重点：减去了左臂的动作空间，固定左臂姿势
```

### Level 5-6（全流程）

```
每步奖励 = 左臂奖励分量 + 右臂奖励分量 + 全局奖励

左臂分量：
  - 接近烧杯 → 抓取成功 → 提升悬空 → 保持稳定
  
右臂分量：
  - 接近烧杯 → 接过成功 → 运到目标 → 放置成功 → 回 home

全局奖励：
  + 递物成功（右爪接到、左爪松开瞬间）：+30
  + 双臂回位：+10
  - 两臂互撞：-10
  - 碰撞桌面：-5
```

## 六、算法选择

| 算法 | 适合 | 不推荐原因 |
|------|------|----------|
| **SAC** (你现在用的) | ✅ 样本效率高，适合连续控制 | 调参比 PPO 多 |
| PPO | 稳定，默认参数就收敛 | 样本效率低，真机训练成本高 |
| HER (Hindsight Experience Replay) | **适合稀疏奖励的递物任务** | 需要改 replay buffer |
| **PPO + HER** | **推荐** | 结合两者优点 |

**建议**：先用 SAC 跑通 Level 1-3，到 Level 4-6 时切换到 **PPO + HER**。
HER 特别适合"抓→放"这类目标条件任务——它会把失败的 episode 重新解释为"接近了目标但没完全对上"，自动产生正样本，大幅提升训练效率。

## 八、关键技术难点

### 1. 双臂碰撞
- 两个 7DOF 臂在同一工作空间操作
- **解法**：在 reward 里加碰撞惩罚，同时限制关节空间范围

### 2. 递物瞬间的协调
- 左臂松开和右臂抓紧的**时序配合**决定了烧杯掉不掉
- **解法**：在递物瞬间留一个短暂 overlap（两爪同时抓 0.1-0.2 秒）

### 3. 力矩控制（OpenArm）
- MJCF torque control 比 position control 难训练
- **解法**：可以先改 MJCF 为 position control 模式跑训练，部署时再改为 torque

### 4. Sim-to-Real 迁移
- 仿真里学到的策略放到真机上会不会掉
- **解法**：域随机化（Domain Randomization）— 在训练时随机化摩擦力、重量、关节阻尼

### 5. 长序列的稀疏奖励
- 全流程 5 步，前面几步的奖励非常稀疏
- **解法**：课程学习本身就是解法。每一步都在为下一步积累经验
