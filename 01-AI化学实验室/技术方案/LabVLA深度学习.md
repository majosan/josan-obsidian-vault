## 一、先搞清楚：Qwen3-VL-4B 本来是干什么的

**Qwen3-VL-4B** 是阿里千问家族的视觉语言模型。它原本的能力：

```
输入                              输出
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[图片]                           → "这是一个实验室场景"
[文字] "画面里有什么？"           → "桌上有烧杯和试管"
```

**本质**：这是一个「看图说话」模型。它学的是**像素 ↔ 语义**的映射。

它内部结构分三块：
```
Qwen3-VL-4B (40亿参数)
├── Vision Encoder (~3亿)    | 把图片变成"视觉特征向量"
├── Qwen LLM (~37亿)         | 理解文字+图片，推理出"意思"
└── LM Head                  | 把"意思"变成文字 token 吐出来
```

关键是最后这个 **LM Head** — 它只擅长输出**文字**（连续 token 序列），不懂"关节角度"、"夹爪开度"这些东西。

---

## 二、LabVLA 对 Qwen 做了什么改造

LabVLA 没有改动 Qwen 本身的结构，而是在它**外面加了一圈**：

```
LabVLA-5B-Base (50亿参数 = 40亿 Qwen + 10亿新东西)
├── Vision Encoder (复用 Qwen 的)     | 照旧：图片→视觉特征
├── Qwen LLM (原封不动)              | 照旧：理解+推理
├── State Encoder ★ 新增              | 把机器人关节角度编码为"状态特征"
├── Action Head / DiT ★ 新增            | 把"意图"变成"动作轨迹"
└── Projection Layer ★ 新增           | 对齐 Qwen 的输出和 DiT 的输入
```

### 新增组件的具体角色

**① State Encoder（状态编码器）**
```
输入：8 维向量 [j1, j2, j3, j4, j5, j6, j7, gripper]
           ↓ 前7个关节弧度 + 夹爪宽度（0~0.08m）
MLP 升维
           ↓
输出：1024 维状态特征（和视觉特征对齐维度）
```
**为什么需要**：Qwen 原来只看图片+文字，不知道"机器臂现在是什么姿势"。State Encoder 就是告诉模型"臂目前的姿势"。

**② Projection Layer（投影层）**
```
Qwen 输出的"意图"（也是 token 序列）
         ↓ 投影层（一个线性变换）
DiT 能理解的"条件信号"
```
**核心作用**：Qwen 输出的 token 维度是 4096，DiT 需要的是 hidden_dim，中间用投影层转换维度。

**③ DiT 动作专家 ← 最核心的新增**

单独放一节讲。

## 三、DiT（Diffusion Transformer）— 动作生成的核心

### 先理解 Diffusion 是什么

想象一下 AI 画画的过程：

```
传统方法：直接预测"猫长什么样"
   ⚠️ 容易模糊（平均了所有猫的特征 → 四不像）

Diffusion 方法：
  第 1 步：纯噪声画面（啥也看不出）
  第 2 步：隐约看到轮廓
  第 3 步：轮廓清晰了
  ...
  第 10 步：清晰的猫
  每一小步模型只做"去一点点噪声"
```

LabVLA 的 DiT 做的是同样的事情，但生成的不是图片，而是**动作轨迹**。

### DiT 的输入输出

```
输入条件（控制生成方向的"提示"）：
  ├── 视觉特征（1024 维）   ← "场景是什么样"
  ├── 文本嵌入              ← "要干什么"
  ├── 状态特征（1024 维）   ← "机械臂当前姿势"
  └── 噪声轨迹 (50,8)       ← 随机数，作为起点

10 步去噪过程：
  每一步读入"当前噪声 + 条件"
  预测"这步要减掉多少噪声"
  输出"更清晰一点的动作"

输出：清晰的 (50,8) 动作轨迹
      50 个时间步 × 8 维动作（7 关节 + 1 夹爪）
```

### 为什么要用 Diffusion 而不是直接预测

核心：**动作的"正确版本"不是唯一的**。

```
抓同一个烧杯，可以有 100 种不同的轨迹：
  A: 从左边绕过去抓
  B: 从右边绕过去抓  
  C: 直接伸手抓

如果直接预测（回归）：
  模型会取平均值 → 得到"向左又向右"的模糊结果 → 鬼畜

如果用 Diffusion（生成式）：
  模型学会"合理的动作分布"
  每次采样出一个清晰的版本
  虽然不是"唯一正确"，但一定是"合理流畅的"
```

**这就是为什么 LabVLA 用 DiT 而不是简单的 MLP 预测动作。** 也是为什么你的 Phase 2 会出现鬼畜——预训练模型没调好，生成的"合理动作"和 MuJoCo 场景不对应，但 DiT 本身的设计逻辑是对的。

## 四、整条数据流 — 逐帧拆解

现在我们可以完整走过一遍**一个推理循环**了：

```
┌── 第 1 步：采集观测 ──────────────────────────────────┐
│                                                         │
│  3 路摄像头 → 3 张 224×224 RGB 图像                      │
│  Franka 关节状态 → 8 维向量                              │
│  文本指令 → "pick up the beaker"                         │
│                                                         │
│  ┌── 视觉编码 ───────────────────────────────────────┐  │
│  │  3 张图各自经过 Vision Encoder (SigLIP)             │  │
│  │          ↓                                          │  │
│  │  每张图 → 196 个 visual token（理解为"图片的碎片"）  │  │
│  │  3 张图 → 588 个 visual token                       │  │
│  │          ↓                                          │  │
│  │  这些 token 和文字 token 一起送入 Qwen               │  │
│  └────────────────────────────────────────────────────┘  │
│                                                         │
│  ┌── 状态编码 ───────────────────────────────────────┐  │
│  │  [j1..j7, gripper] → MLP → 1024 维状态特征         │  │
│  │  状态特征不经过 Qwen，走旁路直接送到 DiT             │  │
│  └────────────────────────────────────────────────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
                           ↓
┌── 第 2 步：Qwen 理解 ──────────────────────────────────┐
│                                                         │
│  Qwen LLM 收到：588 个 visual token + 文本 token        │
│                                                         │
│  它做的事：                                              │
│  ① "这张画面是一个实验室桌面"                             │
│  ② "桌上有烧杯"                                         │
│  ③ "指令说 pick up"                                     │
│  ④ "烧杯在那个位置"                                     │
│                                                         │
│  输出："理解后的意图" → 也是 token 序列                  │
│                                                         │
│  ⚠️ Qwen 此时还不知道"具体关节怎么动"                    │
│  它只负责"高层理解"，不负责"底层动作"                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
                           ↓
┌── 第 3 步：DiT 生成动作 ───────────────────────────────┐
│                                                         │
│  DiT 收到 3 个条件：                                     │
│  ① 视觉特征（来自 Vision Encoder 的原始特征）             │
│  ② Qwen 的"意图"（经投影层后）                           │
│  ③ 状态特征（当前关节姿势）                               │
│                                                         │
│  Step 1: 生成 (50,8) 纯噪声                              │
│  Step 2: 用条件信号引导去噪第1步 → 噪声少一点             │
│  Step 3: 去噪第2步 → 动作轮廓显现                        │
│  ...                                                     │
│  Step 10: 去噪完成 → 清晰的 (50,8) 动作轨迹              │
│                                                         │
│  输出：50 × 8 的 float32 矩阵                            │
│    timestep 0: [0.02, -0.01, 0.05, ..., 0.001, -0.003]  │
│    timestep 1: [0.03, -0.02, 0.04, ..., 0.002, -0.002]  │
│    ...                                                  │
│    timestep 49: [...]                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
                           ↓
┌── 第 4 步：执行动作 ──────────────────────────────────┐
│                                                         │
│  取值：action_chunk[0] → 8 维向量                       │
│  累加：new_joint = current_joint + delta_action         │
│  限幅：检查关节是否超出物理范围                           │
│  执行：送 MuJoCo 或真机                                  │
│  推进 1 个仿真步                                         │
│                                                         │
│  然后重复第 1 步—第 4 步...                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**这是一个推理循环（inference loop）。** 训练时类似，但多一个步骤——算 loss + 反向传播更新权重。

## 五、原子技能（Atomic Skills）— 动作的"语法"

### 原子技能解决什么问题

如果让模型从零学"倒液体"，它需要学的东西太多：
```
识别烧杯 → 算位置 → 规划轨迹 → 抓取 → 移动到目标
  → 调整高度 → 倾斜 → 倾斜角度控制 → 停 → 放回
```

太多子任务混在一起，训练数据很难覆盖。原子技能的思路是**拆解**：

```
"倒液体" = pick(烧杯) + pour(到 B) + place(放回)

每个技能内部的细节（如 pick 的 7 阶段），
由仿真引擎负责规划动作细节，
模型只需要学会"何时调用什么技能"。
```

### 原子技能 vs LabVLA 的关系

**一个关键纠正**：LabVLA 本身不使用原子技能。
- 原子技能是 **RoboGenesis（数据引擎）** 的概念
- RoboGenesis 用原子技能组合 → **自动生成训练数据**
- LabVLA 从数据里学 → 训练好后**直接输出动作轨迹**

```
RoboGenesis 造数据时：
  程序员定义：「pick → pour → place」
  RoboGenesis 自动生成 1000 条具体轨迹
  ↓
  数据给 LabVLA 训练
  ↓
  LabVLA 学完后：
    输入"倒水" → 自动输出完整的动作轨迹
    （不需要显式调用 pick/pour/place，模型内化了）
```

所以：**原子技能是数据生成阶段的工具，不是模型的运行时组件。**

## 六、加入你的触觉 — 多模态融合方案

这是整个链条里你最关心的部分。

### LabVLA 现在的输入

```
3 路 RGB 图像  →  Vision Encoder  →  588 visual tokens
文本指令       →  Tokenizer       →  文本 tokens
机器人状态     →  State Encoder   →  1024 维特征
                           全部拼在一起送入 Qwen
```

### 你要加的东西

```
触觉 124 点  →  VQ-VAE + CNN     →  ?
```

问题是这个 `?` 填什么。

### 方案（之前确认的）

```
触觉 124 点
   ↓ reshape 成 12×12 网格（补 0 到 144 格）
   ↓ 3 通道（Ch1 压力 / Ch2 变化率 / Ch3 振动）
   ↓ CNN 编码（3×3 Conv → 6×6 Conv → 展平）
   ↓
   64 维触觉特征向量

这个 64 维特征怎么接入？
  选择 A：拼到 visual token 后面        → 作为 Qwen 的额外输入
  选择 B：拼到状态特征（State MLP）后面   → 只影响 DiT
  选择 C：独立分支，单独对齐

之前定的是走 A+B 混合（视觉层 + 状态层都加触觉）
```

### 加入后数据流变化

```
原来的：
  Vision Encoder → 588 tokens ────┐
                                    ├→ Qwen LLM → DiT → 动作
  State Encoder → 1024 维 ─────────┘

加入触觉后：
  Vision Encoder → 588 tokens ────┐
                                    ├→ Qwen LLM ──┐
  触觉编码(12×12) → 64 维 ────────┘               │
                                                   ├→ DiT → 动作
  State Encoder → 1024 维 ───────────────────────┘
  (触觉特征还可以再拼一份到这里)
```

## 八、常见误解澄清

### 误解 1："原子技能是模型的一部分"
❌ 原子技能是**数据引擎**的概念，不是模型的概念。模型训练好之后直接输出原始动作轨迹，不需要显式调用原子技能。

### 误解 2："Qwen 被改成了机器人专用版"
❌ Qwen3-VL-4B 代码和权重**完全没改**。LabVLA 只是在它外面加了 State Encoder、Projection Layer、DiT 等新组件。Qwen 依然能做"看图说话"，只是输出被截断+投影，喂给了 DiT 而不是 LM Head。

### 误解 3："DiT 是专门设计的新架构"
❌ DiT（Diffusion Transformer）是 2022 年 OpenAI 提出的成熟技术，最早用于图像生成。LabVLA 把它挪用来生成动作轨迹，有现成实现可复用。

### 误解 4："5B 参数都是动作相关的"
❌ 50 亿参数里 ~40 亿是 Qwen（原封不动，处理视觉+语言），~10 亿是新增的（State Encoder + Projection + DiT）。你的触觉编码器大约再加 0.01 亿参数，极小。
