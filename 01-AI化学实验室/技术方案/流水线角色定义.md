## 一、大图：完整数据流

```
 ┌──────────── 采集层 ────────────┐
 │                                 │
 │  RealSense D435（视频+深度）      │

 │  IMU（手腕倾角/加速度）           │

 └────────────┬───────────────────┘
              │ 原始数据
              ▼
 ┌────────────────────────────────┐
 │     数据预处理管道（我们自己写） │
 │                                │
 │ ① 视频帧 → 裁剪/归一化          │
 │ ② 124点触觉 → VQ-VAE编码压缩   │
 │ ③ IMU → 四元数归一化            │
 │ ④ 时间对齐（多模态同步）         │
 │                                │
 │ 输出：一个时间步的"观测"         │
 │   ├─ 图像特征                   │

              ▼

 │                                │

 │   ├─ 加载 LabVLA-5B-Base 权重  │

 │   └─ 更新权重 → 保存新模型      │
 │                                │
 │  MuJoCo 不参与这一步 ❌         │
 └────────────┬───────────────────┘
              │ 训练好的模型权重
              ▼

 │   └─ 通过 WebSocket 发出       │
 │                                │
 │  MuJoCo                        │
 │   ├─ 接收动作指令               │

 │                                │

 └────────────┬───────────────────┘
              ▼
 🔵 部署：边缘推理卡/服务器
     ├─ LabVLA 框架（推理模式，精简版）

     └─ WebSocket 服务 → 真实机械臂
```

## 二、各组件角色一览

### 🏭 LabVLA 框架

**定位：整条流水线（生产设备 + 流程 + 工人手册）**

| 模式 | 做什么 |
|------|--------|
| **训练模式** | 加载数据 → 喂给模型 → 算 loss → 梯度更新 → 保存权重 |
| **推理模式** | 加载已训练权重 → 收到指令 → 吐出动作 → 通过 WebSocket 发出 |

**需要改代码的地方：**
- 在输入层增加触觉嵌入通道（VQ-VAE 编码结果）
- 其他大部分复用

**部署时：只部署推理模式，不需要完整的训练代码**

---

### 🧠 LabVLA-5B-Base 模型

**定位：50亿参数的**预训练通用机器人操作模型（已训练好的权重文件）

```
LabVLA-5B-Base = 50亿参数
  ├── Vision Encoder  ~0.3B  →  看画面
  ├── Qwen3-VL-4B     ~4B    →  理解+推理（大脑）
  ├── Action Head     ~0.2B  →  生成动作序列
  └── DiT 动作专家    ~0.5B  →  精细动作肌肉记忆
```

**Base 现状：**
- ✅ 认识烧杯、试管等实验室常见物品
- ✅ 掌握抓、放、推、拉等通用操作
- ❌ **没干过化学实验**（不会精确倾倒、不会用移液枪）
- ❌ **没见过触觉手套数据**（124点编码对它全新）

**我们做的事 = 微调，不是从零训练**

---

### 🧠✨ Qwen3-VL-4B

**4B（40亿）参数的视觉语言理解模型，是 LabVLA-5B 的大脑部件**

| 能力 | 对应场景 |
|------|---------|
| 看懂画面 → 识别物体 | "那个圆柱形玻璃是烧杯" |
| 听懂指令 → 理解意图 | "倒水" → 知道要抓→移到→倾斜→倒→放回 |
| 规划步骤 | 拆解为动作序列 |

**微调时不需要改它，它的通用理解能力够用了。**

---

### 🎮 MuJoCo

**定位：纯粹的物理仿真引擎和可视化工具，没有训练能力**

| 在管线中 | 做什么 |
|---------|--------|
| **物理模拟** | 算机械臂按给定轨迹动会怎样（碰撞检测、重力、接触） |
| **可视化** | 渲染画面，让你看着机械臂执行动作 |
| ❌ **不参与训练** | 不用它算梯度，不用它更新模型权重 |
| ❌ **不调用框架/模型** | 反过来，框架调用它 |

**MuJoCo 的角色从数据角度看有两个：**

```
阶段一：验证模型（近期）
  LabVLA(推理) ──动作指令──→ MuJoCo(执行+渲染) ──→ 你看效果

阶段二：数据生成（远期）
  RoboGenesis ──合成轨迹──→ MuJoCo(执行+截图) ──→ 生成训练数据
```

## 三、类比总结

| 现实类比 | 对应 AI 管线组件 |
|---------|----------------|
| **工厂生产线**（传送带、工作站） | LabVLA 框架 |
| **已培训的通用工人** | LabVLA-5B-Base 模型 |
| **工人的大脑**（理解判断） | Qwen3-VL-4B |
| **工人练手的模拟工位**（沙盒，不参与教学） | MuJoCo |
| **工人学新工种**（转行干化学实验） | **微调训练** |
| **在模拟工位考核**（看干得对不对） | MuJoCo 验证 |
| **真机上工** | 部署到真实机械臂 |

## 四、关键纠正点

### ❌ 常见误解

| 说法 | 纠正 |
|------|------|
| "在 MuJoCo 上训练" | 训练在 PyTorch (GPU) 上进行，MuJoCo 只参与验证 |
| "MuJoCo 调用 LabVLA 框架和模型" | 反过来，LabVLA 框架调用 MuJoCo |
| "LabVLA-5B = 5亿参数" | 是 **50亿（Billion）** 参数，模型文件 ~10GB |
| "Qwen3-VL-4B 是框架里的" | 它是 **模型内部的一个部件**（大脑），不是框架 |
