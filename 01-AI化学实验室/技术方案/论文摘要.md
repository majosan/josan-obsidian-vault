# 一、LabVLA 论文精读

> **标题：** Grounding Vision-Language-Action Models in Scientific Laboratories
> **作者：** 浙江大学 + 上海AI Lab + 哈工大
> **链接：** github.com/zjunlp/LabVLA | arXiv:2606.13578

## 1.1 论文要解决什么问题

**现状痛点：** AI 能读文献、写代码、设计实验方案，但**没人把实验方案变成机器人动作**。现有的 VLA 模型只在家庭和桌面场景上训练过，没见过移液枪、离心机、透明液体这些实验室特有元素。

**问题本质：** 数据瓶颈 + 机器人形态多样性。实验室操作数据采集太贵。

## 1.2 核心贡献——两样东西

LabVLA 论文做了两件事，一个是机器人自动做实验的仿真系统，一个是识别照片和指令进而控制机器人的AI模型，分别对应 RoboGenesis 和 LabVLA。

### ① RoboGenesis（仿真数据引擎）
不是模型，是**仿真数据生产线**。基于 NVIDIA Isaac Sim（不是 MuJoCo），三步走：

```
第1步：搭环境
  文字描述 → AI 生成 3D 资产 → 搭建虚拟实验室场景

第2步：生成工作流
  自然语言指令（如"将烧杯A的液体倒入烧杯B"）
    → 分解为原子技能序列（Pick → Pour → Place）
    → 部署到不同机器人（Franka / UR5e / 双臂…）
    → 加域随机化（不同光照/颜色/位置）

第3步：导出数据
  只保留成功的演示 → HDF5 格式 → 对接 ACT / Diffusion Policy 训练
```

**支持 11 种原子技能：** pick / place / pour / shake / stir / press / pressZ / open / close / move / navigate
**支持 13+ 机器人平台：** Franka, UR5e, UR16e, Festo, 双臂 ALOHA…
**四个任务族：** 单臂基础操作 → 多步实验流程 → 双臂协作 → 移动操作

> ⚠️ **注意：** RoboGenesis 基于 **Isaac Sim**（NVIDIA 平台），不是 MuJoCo。我们之前讨论的方案是用 MuJoCo，如果要用 RoboGenesis 则需 Isaac Sim（需要 NVIDIA GPU）。

### ② LabVLA（模型框架）

模型架构：
```
Qwen3-VL-4B（视觉语言基础模型）
    ↓ 理解图像+文本指令
DiT 动作专家（Diffusion Transformer）
    ↓ 输出连续动作
机械臂执行
```

## 1.3 两阶段训练（你的关键知识点）

### 阶段一：FAST 动作 Token 预训练

```
输入：图像 + 文本指令
过程：把"动作"也当成一种"语言"来训练
      Qwen3-VL 先学会理解动作语义
输出：模型知道了"看到这个画面+这个指令=应该做这个动作"
特点：还没学具体动作轨迹，只学"语义对应"
```

**类比：** 先学会看谱（知道哪个符号代表哪个音），但还没练弹琴。

### 阶段二：Flow Matching 后训练 + 知识隔离

```
输入：图像 + 文本指令 + 机器人状态
过程：用 DiT 模型学"如何生成连续流畅的动作轨迹"
      用 Flow Matching 技术把动作做精
      加了知识隔离（stop-gradient）防止动作训练冲乱视觉语言的理解
输出：能输出真实机器人动作的模型
```

**知识隔离是什么：** 
在训练动作专家（DiT）时，视觉语言模型（Qwen3-VL）的部分**不参与反向传播**。这样视觉语言的理解能力不会被动作训练的梯度破坏。

### 注意：LabVLA 论文是"两阶段"不是"三阶段"

之前我跟你说的三阶段（预训练→后训练→微调）是** LabVLA + 你的触觉拓展**后的完整管线。单纯 LabVLA 原文只有两阶段：FAST 预训练 + Flow Matching 后训练。

如果你的项目加上了触觉手套数据，就变成了：
```
① FAST 预训练 → 用 RoboGenesis 仿真数据（纯视觉）
② Flow Matching 后训练 → 仿真数据精炼（纯视觉）
③ 任务微调 → 加你的触觉数据（真实数据，新增触觉通道）
```

## 1.4 对你有用的关键数据

| 指标 | 数据 |
|------|------|
| 基准测试 | LabUtopia |
| In-Distribution 成功率 | 71.1%（最高） |
| Out-of-Distribution 成功率 | 70.0%（最高） |
| 基础模型 | Qwen3-VL-4B（阿里千问系列） |
| 动作专家 | DiT（Diffusion Transformer） |
| 训练数据 | 仿真合成（RoboGenesis） |
| 代码 | ✅ 开源，可直接跑 |

## 1.5 对你的启发

1. **LabVLA 天然适合你** — 同出浙大，它的实验室场景和你的催化剂实验室高度重合
2. **开源代码可直接改** — 你不需要从头写，clone 仓库后加触觉通道即可
3. **注意 Sim 引擎差异** — RoboGenesis 用 Isaac Sim，你之前决定用 MuJoCo。可以用 RoboGenesis 的数据生成理念，但底层引擎换成 MuJoCo
4. **两阶段 vs 三阶段** — 你的触觉数据微调需要自己加，LabVLA 原始管线不包含触觉

# 二、T-Rex 论文精读（Section 3.2 触觉编码部分）

> **标题：** T-Rex: Tactile-Reactive Dexterous Manipulation
> **作者：** UC Berkeley + Stanford + NVIDIA 等（李飞飞、Jim Fan、Ken Goldberg… 豪华阵容）
> **链接：** arXiv:2606.17055 | tactile-rex.github.io

## 2.1 论文全貌（一句话）

> 在 VLA 模型里加入触觉信号，让机器人学会"手感"——知道握多紧、什么时候该松手。

**关键成就：** 相比不加触觉的模型，成功率平均提高 **30%+**。

## 2.2 核心架构——MoT（Mixture-of-Transformer-Experts）

T-Rex 的架构是**三个专家模型一起工作**：

```
输入
  │
  ├── 视觉专家（处理图像，低频 5Hz）
  ├── 语言专家（处理指令）
  └── 触觉专家（处理触觉信号，高频 20Hz）
          │
          ▼
  三个专家的输出融合 → 生成动作
```

**关键是**：视觉和触觉**不同频率**，不能硬塞到一起。T-Rex 用 **异步融合**（asynchronous refinement）解决。

## 2.3 ⭐ Section 3.2 触觉编码——最精华部分

### T-Rex 用了两种触觉传感器

| 传感器类型 | 数据维度 | 编码方式 |
|-----------|---------|---------|
| 6 轴力传感器（指尖） | Fx, Fy, Fz, Tx, Ty, Tz | VQ-VAE（时间） |
| 变形传感器（GelSight） | 图像 | CNN（空间） |

**你的情况：** 你只有一种传感器（124点压力），但数据特性介于两者之间——既有空间分布（124点布局），又有时间序列（连续帧）。所以你需要**两者结合**：CNN（空间）+ VQ-VAE（时间）。

### VQ-VAE 触觉编码器详解

这是 T-Rex 最重要的技术贡献，也是 Section 3.2 的核心：

```
原始触觉数据：连续 16 帧 × 6 维力信号
    │
    ▼
┌── 编码器（Encoder）──────────────┐
│  1D Conv 逐层压缩时间维度          │
│  16帧 → 8帧 → 4帧 → 2帧          │
│  每帧从6维映射到64维              │
└────────┬─────────────────────────┘
         │ 输出：2 × 64 = 128 维特征
         ▼
┌── 向量量化（Vector Quantization）──┐
│  特征 → 查 codebook（1024个原型）   │
│  每个特征找到最接近的原型码         │
│  输出的是原型码的索引，不是原始值    │
│  压缩比 ≈ 180:1                    │
└────────┬─────────────────────────┘
         │ 输出：2 个离散 token
         ▼
┌── 解码器（Decoder）──────────────┐
│  用 token 索引还原触觉信号         │
│  （用于训练编码器）                │
└───────────────────────────────────┘
```

### VQ-VAE 的压缩比

```
原始：16 帧 × 6 维 = 96 个浮点数
编码后：2 个 token（每个是一个 codebook 索引）
等效信息量：96 → 2，约 48:1

你的情况（参考值）：
原始：16 帧 × 124 点 = 1,984 个浮点数
编码后：约 11 个 token
等效信息量：1,984 → 11，约 180:1
```

### 为什么用 VQ-VAE，不用普通 Autoencoder？

| | 普通 AE | VQ-VAE |
|---|---|---|
| 输出 | 连续值 | **离散 token**（索引） |
| 优点 | 信息损失小 | 和 LLM/VLM 的 token 格式统一 |
| 缺点 | 不能直接拼到 Transformer | 训练稍微复杂 |
| 对你的意义 | — | ✅ 输出 token 可直接拼入 LabVLA |

**核心原因：** VLM（视觉语言模型）里所有的东西都是**离散 token**（图像 token、文本 token）。触觉信号也必须变成 token 才能拼进去。VQ-VAE 就是为了把连续信号变成离散 token 而设计的。

## 2.4 异步触觉融合——最关键的设计教训

T-Rex 发现：**触觉和视觉频率不一致，不能同时处理**。

```
视觉：5Hz（每秒5帧）
触觉：20Hz（每秒20帧，是你的4倍）

如果把两者一起训练：
  模型搞混了 "这变化是视觉的" 还是 "这变化是触觉的"
  成功率从 17% → 6%（加了触觉反而更差！）

解决方案：异步融合（asynchronous refinement）
  第 1-6 步去噪（τ_split）：只用视觉规划动作路线
  第 7-10 步去噪：加入触觉做精细调整
```

### 你对这个设计的应用

```
你的触觉 15Hz，视觉 5Hz → 频率差 3 倍
→ 对应异步设计：
  第 1-6 步：视觉规划（纯视觉）
  第 7-10 步：加触觉 token 做微调
```

## 2.5 数据效率结论

T-Rex 还做了数据效率实验，对你这种**真实数据有限**的情况很有价值：

```
只有 10 条演示数据：
  无触觉预训练 → 成功率 ~15%
  有触觉预训练 → 成功率 ~45% （3倍提升！）

有 200 条演示数据：
  无触觉预训练 → 成功率 ~45%
  有触觉预训练 → 成功率 ~65%

结论：触觉训练在数据量少时帮助更大。
      你有 500 条真实数据 → 加触觉可以显著提升效果
```

## 2.6 对你的启发

1. **VQ-VAE + CNN 是你的编码方案** ← T-Rex 验证了这个路线的可行性
2. **你的数据比 T-Rex 更适合 VQ-VAE** — T-Rex 的力传感器只有 6 维，你的 124 点包含了空间信息，VQ-VAE 能学到更丰富的模式
3. **必须做异步融合** — 你的触觉 15Hz vs 视觉 5Hz，差 3 倍，不做异步方案会翻车
4. **触觉数据的价值在低数据量时最大** — 你们只有几百条真实数据，加触觉收益最明显

# 三、RoboGenesis 文档精读

> **网站：** lxj-kai.github.io/robogenesis/v0.5/zh/
> **代码：** github.com/Ylr9933/RoboGenesis
> **位置：** 浙大维护，与 LabVLA 同源

## 3.1 一句话定位

> 自动生成机器人训练数据的仿真工厂——不是模型，是数据生产线。

## 3.2 技术栈

**⚠️ 重要注意：** RoboGenesis 基于 **Isaac Sim 5.1**（NVIDIA 平台），不是 MuJoCo。

| 维度 | RoboGenesis | 你之前的选型 |
|------|------------|------------|
| 仿真引擎 | Isaac Sim 5.1 (PhysX) | MuJoCo + MJX |
| 渲染 | 高保真（光线追踪级） | 简约（物理优先） |
| GPU要求 | ✅ NVIDIA GPU 必须 | ✅ 可跑在 CPU 或任何 GPU |
| 你的适配 | 需要改底层引擎 | 已决定用 MuJoCo |

**对你的含义：** 你可以用 RoboGenesis 的 **数据生成方法论**（原子技能组合、域随机化、成功过滤），但底层引擎换成 MuJoCo。

## 3.3 11 种原子技能详解

这是 RoboGenesis 最核心的东西——把复杂的实验室操作拆解成**可组合的最小单元**：

| 技能 | 做啥 | 阶段数 | 对你的实验室 |
|------|------|--------|------------|
| **pick** | 抓取物体（7阶段） | 7 | ✅ 抓烧杯/试管 |
| **place** | 放置物体（6阶段） | 6 | ✅ 放回架位 |
| **pour** | 倾倒液体（6阶段） | 6 | ✅ 倒液体（催化剂合成核心） |
| **shake** | 摇晃（10阶段） | 10 | ✅ 混合溶液 |
| **stir** | 搅拌（5阶段） | 5 | ✅ 搅拌反应液 |
| **press** | 按压（3阶段） | 3 | ✅ 按移液枪/按钮 |
| **pressZ** | 垂直插入（3-4阶段） | 3-4 | ✅ 插移液枪头/pH计 |
| **open** | 打开（8阶段） | 8 | ⚠️ 开瓶盖/开箱门 |
| **close** | 关闭（3阶段） | 3 | ⚠️ 关瓶盖 |
| **move** | 平移（无固定阶段） | 可变 | ✅ 沿实验台滑动 |
| **navigate** | 导航移动 | 可变 | ⚠️ 移动机器人底座 |

### 对你们的覆盖分析

```
11 种技能中 ✅ 直接可用 6 种：
  pick / place / pour / press / pressZ / move

⚠️ 需要参数调整 2 种：
  shake / stir（催化剂合成的摇晃搅拌，需要调速度/力度参数）

✅ 需要自定义 2 种：
  旋转瓶盖（类似 open 但单手旋转）
  擦拭（清洁/涂抹表面）

⚠️ 暂时不需要：navigate（你们的机械臂是固定的）
```

## 3.4 原子技能的内部结构（以 pick 为例）

每个原子技能是一个**多阶段状态机**：

```
Pick 技能（7个阶段）：
  阶段1: 移动到物体上方预备位置
  阶段2: 定位到物体（视觉伺服）
  阶段3: 下降到抓取高度
  阶段4: 闭合夹爪
  阶段5: 确认抓取成功
  阶段6: 提起物体（回到安全高度）
  阶段7: 结束

参数配置（YAML）：
  Pick:
    approach_height: 0.1m     # 预备高度
    grasp_width: 0.05m        # 张开宽度
    lift_height: 0.15m        # 提起高度
    speed: 0.5                 # 速度倍率
```

## 3.5 域随机化（Domain Randomization）

RoboGenesis 支持**六轴随机化**，让模型在更多变的环境中学会技能：

```
场景随机化：每次生成不同的桌面布局
相机随机化：不同视角、不同分辨率
光照随机化：不同亮度和色温
物体随机化：不同颜色、不同大小
空间随机化：位置偏移 ±5cm、旋转 ±15°
物理随机化：摩擦力、质量轻微变化
```

对你的价值：在仿真里做域随机化，模型到真机时**泛化能力更强**。

## 3.6 数据导出格式

RoboGenesis 导出 **HDF5** 格式，包含：

```
/observations/rgb   ← 多视角 RGB 图像
/observations/depth  ← 深度图  
/observations/state  ← 机器人关节状态
/actions             ← 动作轨迹
/language            ← 文本指令
/subtask_labels      ← 子任务标注
```

## 3.7 对你的启发

1. **方法论可直接复用** — 即使底层引擎换成 MuJoCo，原子技能组合、域随机化、成功过滤的思路不变
2. **8 种操作中 6 种已有直接映射** — 不用从零设计
3. **自定义 2 种技能** — 旋转瓶盖和擦拭，需要设计状态机（参考 open 和 press 的模板）
4. **数据格式参考** — 你的数据导出也可以参考 HDF5 的组织方式
