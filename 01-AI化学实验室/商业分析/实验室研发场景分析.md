## 二、聚焦"材料合成/制备"环节

在新材料研发中，制备方法有很多种。逐一分析"纯自动化"和"纯人"各自的局限：

### 方法1：溶液涂布法（旋涂/刮涂/喷涂/狭缝涂布）

**这是新材料的核心制备手段之一。** 钙钛矿、有机半导体、催化剂层、电子传输层等。

| 维度 | 现状 |
|:----|:------|
| 纯自动化现状 | 工业涂布有固定参数，但**新材料研发中每个配方都需要调参** |
| Chemspeed 能做？ | 可以自动化涂布，但只能按预设参数跑，不会随配方变化自适应 |
| 纯人能做？ | 有经验的实验员能根据"这管溶液看起来/摸起来怎么样"判断涂布参数该怎么调 |
| **问题核心** | 涂布参数（速度、刀高、温度）和浆料流变特性之间存在复杂关系，且每换一个配方就要重新摸索一遍参数 |

**这个环节的"手感"价值：**

> **一个有经验的薄膜科学家，在看到/摸到一个新配方的浆料时，能在5秒内判断："这个粘度偏高，刀高要调高20微米，速度要放慢到原来的70%。"**
>
> 这种判断来自多年的触觉+视觉经验积累。现有自动化系统做不了这个判断——它们只能按设定参数运行。
>
> **你的手套可以做什么？**
> 把"浆料摸起来什么感觉"变成数据，让AI学会这个关联。
> 然后：新配方进来 → 机器人用手套摸一下评估 → AI自动推荐涂布参数 → 涂布。

### 方法2：固态合成法（球磨+烧结）

| 维度 | 现状 |
|:----|:------|
| 纯自动化现状 | Chemspeed 可以做自动称量和球磨，但**开罐评估产物状态**目前需要人 |
| 手工操作 | 球磨后开罐，看颜色、摸粗细、闻气味来判断反应进展 |
| 触觉价值 | 中等。视觉（颜色）+XRD（物相）已经能替代大部分触觉判断 |

### 方法3：溶胶-凝胶法

| 维度 | 现状 |
|:----|:------|
| **纯自动化做不了** | 凝胶化点的判断——溶液突然变稠的那一刻，时间窗口很短，错过了就过了 |
| **纯人做不好** | 人盯着溶液看+偶尔搅一下，效率低且主观差异大 |
| **触觉价值** | ⭐⭐⭐⭐⭐ 凝胶化点可以通过搅拌阻力的突变来判断 → 手套的压力数据精确捕捉这个时刻 → 自动停止 → 比人更及时、更一致 |

## 三、最有说服力的场景推演：高通量薄膜配方优化

### 场景设定

假设某新材料的薄膜器件（钙钛矿太阳能电池、有机光电、催化涂层）需要优化配方。

**传统的高通量做法（Chemspeed能做的极限）：**
1. 自动配制100个不同配方的前驱液
2. 自动在100个基板上沉积薄膜（固定涂布参数）
3. 自动表征100个器件
4. 数据分析 → 找到最优配方

**问题：** 步骤2用固定涂布参数，意味着**所有配方的涂布条件一样**。但实际上配方A和配方B的溶剂比不同、浓度不同、粘度不同，用同样参数涂出来的膜质量天差地别——因为涂抹效果本身就成了一个变量，污染了"配方本身优劣"的判断。

> **这不是假设问题，而是钙钛矿研究中真实存在的痛点。** 任何一个做过钙钛矿的实验员都知道：换了前驱体配比后，旋涂参数必须重新优化，否则性能差异可能90%来自涂布质量而非配方的本征性能。

### 你们方案的价值链

```
高         ↑ 
通          |     你们的目标区：
量          |     可以高通量跑不同配方，
效          |     同时每个人配方都用"最优"的涂布参数
率          |
           |
           |     Chemspeed区：
           |     高通量但涂布参数固定，
           |     不同配方无法公平对比
           |
           |     纯手工区：
           |     可以每人配方调参，但效率低
           |
           └─────────────────────────────→
                    配方灵活度（能否处理不同配方）
```

### 具体操作流

```
Step 1: 初始学习（人机协作）
  工程师戴上手套 → 做5-10个不同配方的涂布
  每个配方：先"摸浆料"评估流变性 → 凭经验设定涂布参数 → 涂布 → 观察膜质量
  手套记录：触觉数据 + 涂布参数 + 膜质量结果

Step 2: AI建立模型
  输入：触觉特征 + 涂布参数 → 输出：预测膜质量
  AI学会：当触觉数据是XXX时，最优涂布参数是YYY

Step 3: 高通量自动运行
  新配方浆料进入 → 机器人用夹爪（带触觉传感）评估浆料 → AI推荐涂布参数 → 自动涂布 → 表征
  → 100个配方，每个都用"最适合它"的参数跑

Step 4: 公平对比
  所有配方的膜质量差异 → 反映了配方的本征性能差异 → 而不是涂布质量的差异
  科学家得到真实可信的配方排名
```

### 为什么Chemspeed做不到？

Chemspeed的Rumble平台能做自动化浆料配制和涂布，但它是 **"同参数批量执行"**——要么给100个配方用同一套参数，要么给每个配方单独编程参数（但需要工程师介入）。

你们的方案是 **"自适应的——浆料不同，参数自动调"**。这个自适应能力来自：
1. 触觉传感（手套阵列）评估浆料状态 → 你们独有
2. AI学会触觉-参数-质量的关联 → 可以训练

### 为什么纯人手做不到？

人手可以给每个配方单独调参，但一天做5-10个就是极限了。而且人的判断有主观差异："今天手感觉得这个配方粘稠度不错"——但明天手感可能就变了。

## 四、对投资人的叙事（推敲过的版本）

> **"新材料研发中有一个被长期忽视的问题：当你用一种固定工艺去比较不同配方时，工艺和配方在结果上是耦合的。你不知道性能差异是来自配方本身，还是来自工艺不匹配。**
>
> **这个问题在薄膜材料研发中尤其严重——钙钛矿、催化涂层、电子传输层，都需要把前驱液涂成薄膜。但不同配方的浆料流变性天差地别。**
>
> **现在实验室怎么解决？靠人。有经验的实验员会"摸一下浆料"，根据手感调整涂布参数。但人一天只能做5-10个。**
>
> **Chemspeed 可以自动涂100个，但参数是死的。用死参数跑100个不同配方的浆料，出来的结果没法公平比较。**
>
> **我们的方案：用触觉手套把『手感』变成数据，AI学会『这种手感→用这些参数』的映射。然后机器人自动做100个配方，每个配方都用最适合它的参数跑。**
>
> **这是 Chemspeed 做不了的事，也是纯手工效率太低的事。这就是我们的人机协作价值。"**
