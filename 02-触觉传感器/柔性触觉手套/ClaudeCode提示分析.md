## 📋 任务总览

你需要写一套 Python 分析脚本，处理触觉手套测试产生的 JSON 数据，输出结构化分析报告。

**输入**：多个测试 JSON 文件（格式见下方说明）
**输出**：每个测试项的分析结果（数值 + 通过/不通过判定）+ 可视化图表

---

## 📁 数据格式说明

每个 JSON 文件是手套传感器数据。你的脚本应该兼容以下两种可能格式：

### 格式 A（单帧）
```json
{
  "points": {
    "P1": 0, "P2": 15, "P3": 42, ...
  },
  "timestamp": 1720000000000
}
```
124 个点，键名可能是 `P1-P124` 或类似命名。

### 格式 B（多帧/时间序列）
```json
{
  "frames": [
    {"timestamp": ..., "points": {"P1": 0, "P2": 15, ...}},
    {"timestamp": ..., "points": {"P1": 2, "P2": 18, ...}},
    ...
  ]
}
```

**你的脚本第一步应该是自动探测数据格式**，然后统一处理。对多帧数据，按时间序列分析；对单帧数据，按静态快照分析。

---

## 🔬 需要实现的 6 个分析模块

### 模块 1：空手基线稳定性分析
**输入**：30 秒连续采集的 JSON（多帧格式）
**处理逻辑**：
1. 对 124 个点各计算：min / max / mean / std
2. 计算每个点的波动范围 = max - min
3. 标记波动 > 5 的异常点
4. 检测持续漂移：用线性回归拟合每个点的时间序列，斜率绝对值 > 0.1/秒 → 标记漂移
**输出**：
- 124 点的统计表（mean 列、std 列、波动范围列）
- 异常点列表（编号、波动值、是否漂移）
- 判定：波动 ≤5 的点数 / 124，≥ 110 个 → 🟢通过
- 124 点基线均值热力图（映射到手套位置）

### 模块 2：单点加压响应曲线分析
**输入**：对同一个点 Px，多组 JSON 文件——"Px_空载.json"、"Px_5g_r1.json" 到 "Px_500g_r5.json"，每级砝码 5 次重复
**处理逻辑**：
1. 对每个重量级别，提取该点的读数，计算 mean、std、CV
2. 检查单调性：重量递增时 mean 是否严格递增（允许相邻 ≤3 的微小波动）
3. 计算有效量程：500g 时 mean - 空载时 mean
4. 计算最小可检测重量：第一级使 mean > 空载 mean + 7 的砝码重量
5. 检测饱和：如果某级砝码的 mean 与下一级差值 < 3，且后续连续 < 3，标记此处为饱和点
**输出**：
- 每个选点的「重量-响应曲线」图（x=砝码g，y=读数，带 error bar 用 std）
- 汇总表：每个点的单调性、CV(max)、有效量程、最小触发重量、饱和重量
- 判定：8/8 单调递增？全部 CV ≤ 15%？全部量程 ≥ 150？

### 模块 3：连续重复放置一致性分析
**输入**：同一个点同一重量连续 20 次的 JSON 文件
**处理逻辑**：
1. 提取该点 20 次读数
2. 计算 mean、std、CV
3. 画时序图：x=次数，y=读数，看有无漂移趋势
**输出**：
- CV 值 + 判定（≤10% → 🟢）
- 重复性时序图

### 模块 4：跨点差异分析
**输入**：15 个采样点在 50g/100g/200g 各一次的 JSON 文件或汇总表
**处理逻辑**：
1. 对每个重量级别，列出 15 个点的读数值
2. 计算 min / max / mean / std / CV（跨点）
3. 标记最大偏离点（读数离均值最远的点）
4. 计算每个点的"校准因子" = 全局均值 / 该点读数（用于后续软件校准）
5. 按区域分组（指尖 vs 中节 vs 掌心），看有无区域趋势
**输出**：
- 15 点对比表
- 3 个重量级别下的校准因子表
- 区域趋势分析（指尖平均 vs 掌心平均）

### 模块 5：握持动作分析（水瓶/烧杯/试管）
**输入**：多帧 JSON（包含多次握持-放松循环，每帧有 124 点读数）
**处理逻辑**：
1. **自动分割握持-放松段**：找出高活性帧（全手读数总和超过基线总和 + 阈值）→ 定义为"握持段"；低活性帧 → "放松段"
2. 对握持段求平均热力图：每帧的 124 点取帧平均
3. 对放松段求平均热力图
4. 提取握持段每帧的 Top 10 高压点（位置+读数）
5. 检查"模式稳定性"：多次握持之间，Top 10 点位置的重叠率（如 5 次握持有几个点的 Top 10 是重合的）
6. 计算握持 vs 放松的区分度：握持段全手读数总和 mean - 放松段全手总和 mean
**输出**：
- 握持段和放松段的对比热力图（并排显示）
- Top 10 高压点稳定性表
- 区分度数值 + 判定（≥15 → 🟢）

### 模块 6：弯曲干扰分析
**输入**：多帧 JSON（手掌悬空、摊开→握拳→展开，多次循环）
**处理逻辑**：
1. 对 124 个点各计算：全时段 max - min
2. 标记"假阳性点"：该点在握拳阶段读数 > 阈值 7（即被认为有压力，实际上没有接触）
3. 统计假阳性点的数量和最大读数
4. 按区域汇总假阳性分布（手指区 vs 掌心区）
**输出**：
- 假阳性点列表（编号、区域、最大假阳性读数）
- 假阳性区域热力图
- 汇总：假阳性点数 / 124、最大假阳性读数

## 📊 统一输出格式

所有分析完成后，生成一个汇总 HTML 报告页面，包含：

1. **总结表**：每个测试项的判定结果（🟢/⚠️/🔴）
2. **每项测试的详细分析图表和数值**
3. **一个总体判定**：手套是否达到"数据可用于 VLA 训练"的最低标准

判定标准参考 `test-protocol-v1.md` 的汇总表。

## 🛠 技术要求

- 语言：Python 3.10+
- 可视化：matplotlib + seaborn（热力图）
- 数据处理：numpy + pandas
- 报告生成：HTML（可直接浏览器打开）
- 代码放在 `analysis/` 目录下
- 每个模块独立一个 `.py` 文件
- 主入口 `analyze.py`，按文件夹路径读取所有测试 JSON，自动识别并运行对应模块

---

## ⚠️ 注意事项

1. **跨点一致性不作为硬性判定**——柔性织物天生做不到。模块 4 仅做定量记录，不做通过/不通过判定。
2. **阈值 = 7**：读数 < 7 视为无压力。
3. **124 个点的命名/映射**需要你解析实际 JSON 结构后确认。
4. 如果某种测试数据缺失，脚本应跳过该模块并标注"N/A"，不要报错。
5. 所有图表自动保存到 `analysis/output/` 目录。
