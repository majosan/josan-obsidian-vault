# 触觉手套验收测试方案 V1

> 制定日期：2026-07-16
> 测试对象：两副手套样品（各 124 个传感点）
> 已知约束：阈值 < 7（低于 7 忽略）、柔性织物跨点一致性差（接受逐点校准）、1g 精度砝码

---

## 测试逻辑

不追求跨点一致性（柔性织物做不到），**核心验证三条**：

1. **单点可靠性**：同一个点在相同压力下读数稳定、递增方向正确
2. **空间可用性**：不同点响应特性差多少（定量记录，用于后续校准策略）
3. **功能可用性**：实际握持物体时数据分布是否合理（是否能用于 LabVLA 数据采集）

> ⚠️ **不设统一跨点一致性硬指标**。跨点差异是已知固有问题，不参与合格判定。

## ⚠️ 测试前必做

### 0. 称量标准物体
用电子秤称量并记录：
- 500ml 矿泉水瓶（满水）实际重量：___ g
- 250ml 烧杯（满水）实际重量：___ g
- 100ml 量筒/试管（满水）实际重量：___ g

### 0.1 穿戴标准化
每次戴上手套后：
- 指尖尽量顶到手套末端
- 手腕魔术贴/松紧带收紧到一致松紧度
- **戴上后不要频繁脱下再戴上**——穿脱会改变织物拉伸状态，影响基线
- 如果中途脱下，重新戴上后需重新记录基线

## 第一轮：单点可靠性测试

**目的**：验证每个传感点在已知压力下的行为——稳定、递增、可重复。

### 测试 1.1 空手基线稳定性

**操作**：
1. 戴好手套，手掌自然摊开放在桌面（不施加任何压力）
2. 保持不动 30 秒
3. 导出一次 JSON 数据

**记录**：
- 统计 124 个点的 min / max / mean / std
- 标记出波动 > ±5 的异常点

**合格标准 🟢**：
- ≥ 110 个点（90%）波动在 ±5 以内
- 无点出现持续漂移（一直往上或往下走）

**不合格 ❌**：
- 超过 20 个点波动 > ±10
- 或大量点出现持续漂移

### 测试 1.2 单点加压响应曲线

**选点**：从 124 个点中选 **8 个代表点**，覆盖不同区域：

| 编号 | 位置 | 说明 |
|------|------|------|
| P1 | 拇指指尖 | 最常用操作点 |
| P2 | 食指中节 | 握持主力点 |
| P3 | 中指中节 | 握持主力点 |
| P4 | 无名指中节 | 辅助握持 |
| P5 | 小指中节 | 辅助握持 |
| P6 | 拇指根部（大鱼际） | 掌心主力区 |
| P7 | 掌心中心 | 全掌接触代表 |
| P8 | 掌边缘（小鱼际） | 边缘代表 |

**操作**（对每个选点重复）：
1. 手掌平摊桌面，找到当前测试点的物理位置
2. 先记一次空载读数（不放砝码）
3. 垫上塑料片，逐个加砝码：**5g → 10g → 20g → 50g → 100g → 200g → 500g**
4. 每加一次砝码，等待 2 秒读数稳定，记下该点的读数
5. 每级砝码重复 **5 次**（取下来重新放）
6. 导出 JSON → 命名 `A_singlepoint_P1_fullrange.json`（可包含该点所有数据）

**记录表**（每个选点一张）：

| 砝码 | 第1次 | 第2次 | 第3次 | 第4次 | 第5次 | 均值 | CV |
|------|-------|-------|-------|-------|-------|------|-----|
| 空载 | | | | | | | |
| 5g | | | | | | | |
| 10g | | | | | | | |
| 20g | | | | | | | |
| 50g | | | | | | | |
| 100g | | | | | | | |
| 200g | | | | | | | |
| 500g | | | | | | | |

> CV = std / mean × 100%

**合格标准 🟢**：
- 每个点的响应是**单调递增**的（砝码加重 → 读数升，不会出现倒挂）
- 每级砝码 5 次重复 CV ≤ **15%**
- 能从空载到 500g 覆盖至少 **150 个单位**的变化（如从 8→158，差值 ≥ 150）
- 在 50g 以内就能看到明显区分于空载的读数（差值 ≥ 10）

**关注 ⚠️**：
- 如果 5g/10g 始终触发不了（一直在 7 以下），记录"最小可检测重量" = 实际触发的最小砝码重量
- 如果在 500g 前就已经饱和（再加读数不变），记录"饱和重量"

### 测试 1.3 单点重复放置（更严格的一致性）

**操作**（选 P2 和 P7 两个点）：
1. 用 100g 砝码，在同一个点上**连续放置 20 次**
2. 每次放—记—取—放
3. 计算 20 次的 CV

**合格标准 🟢**：
- CV ≤ 10%（两次都满足）

**不合格 ❌**：
- CV > 20%（该点一致性不可靠）

## 第二轮：跨点差异定量记录

**目的**：量化"同样压力压不同点差异多少"——不是判定合格与否，是**为后续的逐点校准策略提供参数**。

### 测试 2.1 同重量跨点扫描

**操作**：
1. 选 3 个重量：**50g、100g、200g**
2. 对每个重量，分别在以下 **15 个采样点**各放置一次，记录读数：

采样点分布（从 124 个中按区域均匀抽取）：

| 区域 | 采样点编号 |
|------|-----------|
| 拇指 | 2 个（指尖 + 根部） |
| 食指 | 2 个（指尖 + 中节） |
| 中指 | 2 个（指尖 + 中节） |
| 无名指 | 2 个（指尖 + 中节） |
| 小指 | 2 个（指尖 + 中节） |
| 掌心 | 5 个（中心 + 四角） |

**记录表**：

| 点位 | 区域 | 50g 读数 | 100g 读数 | 200g 读数 |
|------|------|---------|----------|----------|
| 1 | 拇指尖 | | | |
| 2 | 拇指根 | | | |
| ... | ... | | | |
| 15 | 掌心角 | | | |

**分析**（无需现场判定，数据给我分析）：
- 同重量下 15 点的 min / max / mean / std
- 每点的"校准因子"：假设所有点的目标是对齐到均值，需乘的倍数
- 是否有系统性偏差模式（如：指尖普遍比掌心灵敏？或反过来？）

## 第三轮：功能性握持测试

**目的**：验证手套在实际操作中能否产出"可理解"的压力分布数据。

### 测试 3.1 标准握持——水瓶

**操作**：
1. 戴好手套，自然握住 500ml 满水矿泉水瓶
2. 保持握持姿势稳定 5 秒
3. 放松 3 秒
4. 重复步骤 2-3 共 **5 次**
5. 导出 JSON 文件，标注每次握持的时间段

**参考：好的水瓶握持数据应该长什么样**

```
压力分布特征（理想）：
- 🔴 高压区：拇指指腹区域（拇指负责对抗另外四指，压力最大）
- 🟠 中高压区：食指 + 中指的中节指腹（主握力指）
- 🟡 中压区：无名指 + 小指指腹（辅助握持）
- 🟢 低压区：拇指根部/大鱼际（手掌包裹接触）
- ⚪ 极低或无：指尖末端（握瓶时指尖不一定接触）、手背区域

关键特征：
- 压力分布呈"C 形"包裹模式——拇指一侧 + 四指一侧，中间掌心较空
- 数据和拇指-四指之间的间距应负相关（瓶子越粗，掌心接触越少）
- 5 次握持的压力分布形状应相似（图案可辨认，不是每次都换一张脸）
```

**记录**：
- 握持阶段每帧 124 点中前 10 个最高读数点及其位置
- 5 次握持中"最高压力点位置"是否一致（同一个点？还是每次都变？）
- 握持 vs 放松之间读数差是否明显（差 ≥ 15？还是模糊不清？）

**合格标准 🟢**：
- 压力分布有明显空间结构（不是所有 124 点随机亮）
- 手指区域的压力 > 掌心区域的随机点压力
- 5 次握持的热力图**肉眼可辨是同一个动作**
- 握持 vs 放松的差值足够大，能明确区分两个状态

**不合格 ❌**：
- 压力分布无规律，看起来像随机噪点
- 5 次握持热力图各不一样
- 握持和放松的读数差 < 10（区分不了状态）

### 测试 3.2 标准握持——烧杯

**操作**：同上，换用 250ml 满水烧杯（烧杯口径比水瓶粗，握持方式不同）

**参考特征**：
- 拇指 + 食指/中指压力较均衡（烧杯需要更多手指参与支撑）
- 相比水瓶，掌心区域接触可能更多（因为烧杯直径更大）
- 食指和中指的指尖区域可能会有读数（如果采用"夹持"方式）

### 测试 3.3 标准握持——试管/量筒

**操作**：同上，换用 100ml 试管或量筒

**参考特征**：
- 压力集中在 2-3 个手指的指尖区域（精细操作）
- 整体压力值低于握水瓶（不需要那么大力）
- 应该是 3 个测试中压力读数最小的

### 测试 3.4 手指按压桌面

**操作**：
1. 手掌悬空，单独用食指按压桌面，力度由轻到重再到轻（约 10 秒）
2. 同样用拇指按压桌面
3. 同样用中指按压桌面

**记录**：按压过程中目标手指区域的读数变化曲线，其他区域是否有串扰

**合格标准 🟢**：
- 目标手指区域读数有清晰升降
- 其他手指区域的串扰 ≤ 目标区域最大读数的 20%

### 测试 3.5 弯曲干扰测试

**目的**：这是柔性织物的核心痛点——手指弯曲会导致基线漂移。

**操作**：
1. 手悬空不接触任何物体
2. 手掌完全摊开 → 保持 3 秒 → 慢慢握拳 → 保持 3 秒 → 慢慢展开
3. 重复 5 次
4. 导出全程数据

**记录**：
- 每个点在握拳时读数的最大变化值
- 有多少个点因为弯曲导致读数超过阈值（假阳性触发）

**关注 ⚠️**（非硬性指标，但需要知道）：
- 假阳性触发点数量：___ 个（124 个中）
- 假阳性触发点的最大读数：___
- 这些假阳性点集中在哪些区域？

> 这个数据决定了**是否需要弯曲补偿算法**、以及补偿的难度。

## 第四轮：双样品对比

**目的**：评估两副手套之间的一致性——同一个人做同一动作，两副手套数据是否接近？

**操作**：
1. 用样品 A 完成「测试 1.3 单点重复（P2, 100g）」和「测试 3.1 握水瓶」
2. 脱下样品 A，戴上样品 B
3. 重复步骤 1
4. 对比两组数据

**记录**：
- P2 点 100g 下两副手套的均值差：A=___ B=___ 差值=___ %
- 握水瓶热力图对比：两副手套的压力分布形状是否相似？

**判定**：
- 🟢 好：P2 均值差 < 30%，握瓶热力图形态相似
- ⚠️ 可接受：P2 均值差 30-60%，热力图形态能看出相似性
- 🔴 需改进：差距更大，说明两副手套的批量一致性需要从工艺层面改进

## 额外说明：最小可检测重量

你提到"现在需要用力才能触发"，这个"力"是多少现在不知道。

**测试 1.2 会直接回答这个问题**——看 5g/10g/20g 哪一级能稳定触发（读数 > 阈值 7 + 稳定不飘移），那就是你的手套的**最小可检测力**。

如果你发现需要 50g 以上才能触发，而你的应用场景（比如精细的试管操作）力通常小于 50g，那就要回去**调低阈值或调高灵敏度**。
