# 人机协同多模态传感架构讨论

> 创建日期：2026-07-21
> 下次讨论时先读此文档

---

## 一、核心矛盾

LabVLA-5B 原生设计为**2 指平行夹爪**（8 维状态空间），而我们拥有的是**5 指织物触觉手套**（124 维触觉 + 人手关节）。两者的末端执行器和传感模态不一致，导致数据难以直接对齐。

---

## 二、三种传感场景

### 场景 1：夹爪 + 实时热力图（当前方案 — 过渡）

- MuJoCo 中原有夹爪场景不变
- 旁边开 matplotlib 窗口实时显示 12×12 触觉网格
- 夹爪抓物体时同步显示"如果是手会感受到的触觉分布"
- 目的：可视化验证，不解决根本问题
- **阶段**：当前执行

### 场景 2：灵巧手戴手套（自主操作 — 数据采集）

- 灵巧手（如因时 RH56BFX）穿上织物触觉手套
- 灵巧手自己执行抓取/操作，手套采集真实触觉
- 触觉数据 = 机器人真实触觉，训练端到端模型
- **差异化**：目前市面上缺乏织物触觉灵巧手的训练数据集
- **风险**：手套手指尺寸 vs 灵巧手手指尺寸是否匹配（待验证）

### 场景 3：人 + 灵巧手协同（双向触觉人机协作 — 最终目标）

- 人戴手套操作物体（拿杯子、递工具）
- 灵巧手带/不带手套响应（递滴管、接物体）
- 双向触觉 + 视觉 + 关节状态 → 多模态意图理解
- **核心价值**：通过触觉理解人机协作意图，市面上几乎无人做

---

## 三、两种数据采集方案

| | 方案 α（人→机器人单向） | 方案 β（双向触觉） |
|---|---|---|
| 人戴手套 | ✅ | ✅ |
| 灵巧手戴手套 | ❌ | ✅ |
| 触觉数据 | 人感知物体，机器人看+听 | 双向感知，人和机器人互相知道触觉 |
| 数据复杂度 | 低 | 高 |
| 安全性 | 灵巧手碰到人时无触觉反馈 | 灵巧手碰到人手知道收力 |
| 差异化 | 一般 | 强 — 全球首个织物触觉人机协同系统 |
