## 1. 项目概述

### 1.1 项目定位

打造**高通量催化剂合成无人化学实验平台**。LLM 自动生成实验方案 → 具身智能机器人自动执行 → 表征设备自动分析结果。

类比：晶泰科技（生物制药领域）的化学实验版本。

### 1.2 核心目标

- **近期（0-6 月）**：跑通化学实验室 8 种核心原子操作的 VLA 控制闭环
- **中期（6-12 月）**：灵巧手戴织物触觉手套，自主完成全流程化学实验
- **远期**：人+机器人双向触觉协同操作（全球首个织物触觉人机协同系统）

### 1.3 物理实验室现状

| 项目 | 详情 |
|:----|:------|
| 实验室地点 | 浙江大学 |
| 研究方向 | 催化剂合成高通量实验 |
| 已就绪 | 表征设备 + 实验平台已搭建完成 |
| 待补充 | 具身智能机器人自动化层 |
| 触觉手套 | ✅ 自研 124 点织物压力传感手套，2 套样品已缝制完成 |
| VLA 框架 | ✅ LabVLA 已本地部署，Phase 1&2 跑通 |
| 团队配置 | 1 人（Josan）+ AI 总工 + Claude Code |

```
[人戴手套操作] → 124 点压力 + IMU 数据 → 触觉编码器 → LabVLA 观测空间

                                                          ↓
```

## 8. 软件需求：LabVLA 框架对接

### 8.1 LabVLA 框架概况

| 项目 | 详情 |
|:----|:-----|
| 框架名称 | LabVLA（ZJU/上海 AI Lab） |
| 模型基座 | Qwen3-VL-4B (VLM) + DiT 动作专家 |
| 模型大小 | LabVLA-5B-Base (~10GB, 50 亿参数) |
| 当前状态 | ✅ 已本地部署，Phase 1&2 跑通 |
| 通信协议 | **WebSocket** + msgpack 编码 |
| 动作空间 | **(50, 8) float32 delta action**（8 维：7 关节 + 1 夹爪） |
| 观测空间 | 3×224×224 RGB + 关节状态（8 维） |
| 推理性能 | HM (RTX 4060 Ti): **~194ms/step**（4-bit 量化） |

### 8.2 观测空间改造要求（增加触觉通道）

| 改造项 | 说明 | 优先级 |
|:------|:-----|:------|
| 触觉数据输入 | 124 维 → 12×12 网格 → 3 通道（压力/变化率/振动） | **P0** |
| 编码器 | CNN 或 VQ-VAE 编码触觉网格 → 16-32 维特征 | P1 |
| 数据注入点 | 拼接至 joint_state 后的观测向量 | **P0** |
| IMU 数据 | 9 维（加速度+陀螺仪+磁力计） → MLP 编码 → 拼入融合层 | P2 |

### 8.3 通信接口要求

| 要求 | 规格 | 说明 |
|:----|:-----|:-----|
| 控制频率 | ≥ 10 Hz（目标 30 Hz） | 当前 ~194ms（5Hz），需优化 |
| 通信延迟 | ≤ 200ms | WebSocket 内网通信 |
| 数据格式 | msgpack 编码 | 兼容现有 LabVLA schema |
| 帧同步 | 触觉 + 视觉 + 关节状态时间戳对齐 | 多传感器同步 |

### 8.4 动作空间映射（灵巧手适配）

| 当前 LabVLA 动作 | 映射到灵巧手 | 说明 |
|:----------------|:------------|:-----|
| 8 维：7 关节 + 1 夹爪 | → 6+2 DOF (因时 RH56BFX) | 夹爪维度映射为**全部手指同步开合** |
| 连续值 [-1, 1] | → 归一化到灵巧手关节范围 | 线性映射 + 关节限位保护 |
| delta mode（增量） | → 增量控制 | 保持兼容 |

**特别注意**：灵巧手的 6+2 DOF 远多于夹爪的 1 DOF，初期过渡方案可以：
1. 用 LabVLA 的夹爪指令控制**所有手指同步开合**
2. 单个手指的独立控制留到后续微调阶段自建数据管道

## 9. 软件需求：触觉手套数据接入

### 9.1 自研织物手套规格

| 参数 | 规格 |
|:----|:-----|
| 传感原理 | 三层织物：镀银纱线（电极）+ 压敏尼龙/碳黑纱线 + 镀银纱线（电极） |
| 传感阵列 | 124 点（手指 60 + 手掌 64） |
| 采集板 | 复用 32×32 FPC 板（1024 点），仅接 124 点 |
| 输出接口 | USB 串口，JSON 格式 |
| 帧率 | 15 fps |
| ADC | 12-bit（0-255） |
| 最小可检测力 | ~5g（暂估，待校准） |
| 已知问题 | 跨点一致性差（接受逐点校准方案） |

### 9.2 数据编码管线

```
125 点手套原始数据（124 压力 + 1 时间戳）
    ↓
12×12 网格映射 (glove_grid_mapper.py)
    ↓
3 通道编码 (Ch1:压力值 / Ch2:变化率 / Ch3:振动特征)
    ↓
VQ-VAE 时间编码 (16 帧压缩) + CNN 空间编码 (网格特征提取)
    ↓
16-32 维触觉特征向量 → 拼入 LabVLA 观测空间
```

## 11. 系统集成要求

### 11.1 硬件集成架构

```
[上位机 / 工控机]
    ├── USB 3.0 ── Intel RealSense D435 × 2-3
    ├── USB ────── 手套采集板（串口 JSON）
    ├── Ethernet ── 轮式底盘（ROS2）
    ├── Ethernet ── 协作机械臂（ROS2）
    ├── RS485 ──── 灵巧手（或经过机械臂中继）
    └── PCIe ───── GPU 推理卡
```

### 11.2 时间同步要求

| 传感器 | 数据 | 频率 | 同步方式 |
|:------|:----|:----:|:--------|
| 触觉手套 | 124 维压力 | 15 Hz | 统一时间戳（工控机时钟） |
| 深度相机 | RGB-D 帧 | 30 Hz | 硬件触发器或软件对齐 |
| 机械臂关节 | 7 关节角度 | 100 Hz | ROS2 时间戳 |
| 灵巧手关节 | 6+2 DOF | 50 Hz | ROS2 时间戳 |
| IMU（手套上） | 9 维 | 100 Hz | 统一时间戳 |

### 11.3 ROS2 集成

所有硬件模块统一通过 **ROS2 Humble** 进行集成：

- `/camera/rgb` — RGB 图像话题
- `/camera/depth` — 深度图像话题
- `/glove/tactile` — 触觉数据话题（124 维 float32）
- `/glove/imu` — IMU 数据话题
- `/arm/joint_states` — 机械臂关节状态话题
- `/hand/joint_commands` — 灵巧手控制指令话题
- `/hand/joint_states` — 灵巧手反馈话题

| 验收项 | 标准 | 方法 |

| 全系统延迟 | 端到端 ≤ 500ms | LabVLA 推理 → 灵巧手动作 |

### 12.2 软件验收

| 验收项 | 标准 | 方法 |
|:------|:----|:-----|

| 数据同步 | 触觉+视觉+关节 <= 1 帧偏差 | 时间戳比对 |
